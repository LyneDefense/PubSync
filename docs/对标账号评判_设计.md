# 对标分析（博主诊断 + 找差距）+ 合规共享引擎 —— 方案设计

> 状态：**设计稿**（2026-06，待实现）。取代 [[找对标_泛搜索流程设计]]（泛搜索已砍）。
> 关联：[[采集模块设计]]、[[阶段B_蒸馏选材改造设计]]、[[商业模式与定位]]。
> 一句话：**砍掉泛搜索**，只留**精确搜索（名字/ID）**建库；新增独立子模块 **「对标分析」**，内含 **博主诊断**（诊断任意一个号——别人的对标博主或自己的账号，出 **硬实力 × 软实力 × 合规** 三区报告）和 **找差距**（我 vs 对标，下一轮）。同一套**合规引擎**复用到 博主诊断 与 蒸馏输出闸门。

---

## 0. 为什么砍泛搜索

泛搜索（领域 → AI 推细分领域 → 关键词召回 → 打分 → 候选）**结构性**给不出价值：

1. **采料口错**：搜账号名只命中"把词写进昵称/简介"的小号；真正的大号人设更宽或主动**藏限流词**（黑话/截图/私信），关键词搜不到。香港保险实测候选全是 2～1573 粉的小号。
2. **相关 ≠ 值得对标**：相关性奖励"够纯"，越纯越是小销售号。
3. **平台能力缺失**：TikHub 对小红书只有**扁平关键词搜索 + 单号抓取**，没有推荐/同类账号/类目榜单图谱。大号是被**推荐 + 人滚雪球浏览**找出来的，这能力我们零接入 →"找"这件事用户+小红书 App 远强于我们。
4. **找相似也跑偏**：会让 AI 凭空造抽象细分领域，不吃种子真实内容。

**结论**：发现交给用户+平台，我们的不对称优势在 **判 / 蒸**。

### 砍 / 留
- **删**：`benchmark_discovery` 的 `flow/recall/vetting/querygen`、`/benchmark/discovery/*` 端点、discovery 会话任务；前端泛搜索页 + 找相似 tab。
- **删**：单独的"诊断我的账号"页（并入 博主诊断·自己模式）。
- **留用**：搜索（`blogger_distillation/search`）、单号抓取、`account_audit`（演进为 博主诊断）、`compliance`（升级为共享引擎）。

## 1. 模块结构与流程

### 1.1 三块，职责不互相污染
| 模块 | 干什么 | 备注 |
|---|---|---|
| **对标库 / 博主资产** | 管理：加博主、采集笔记、蒸馏 Skill | **纯资产管理，里面没有评判入口** |
| **对标分析（新独立子模块）** | **博主诊断**（他人/自己）+ **找差距**（下一轮） | "分析判断"类动作集中在此 |
| 精确搜索 | 名字/ID 找一个号 → 基础资料预览 → 加入对标库 | **搜索不评判、不打分** |

### 1.2 全流程
1. **精确搜索**（名字/ID）→ 结果是**基础资料预览卡**（粉丝/总赞藏/笔记数/简介/几个标题）→「加入对标库」。**到此为止，不评判。**
2. 在 **博主资产** 里采集、蒸馏（与诊断无关，可独立进行）。
3. 想评判时去 **对标分析 → 博主诊断**：选一个目标（对标库里的博主，或"我的账号"）+ 填**对标意图** → 系统**确保有 ≥20 条笔记**（已采过 → 增量采集；没采过 → 默认帮采 20 条）→ 跑三区诊断 → 决策卡。
4. 想看差距 → **对标分析 → 找差距**（下一轮）。

> **关键**：诊断**必须基于真实采集的笔记**（评内容就得有内容）。所以诊断**不在搜索时跑**（搜索时没内容、且会平白产生采集开销），而是在对标分析里**用户主动发起**。一次诊断 ≈ 一次 ≥20 条采集的成本。

## 2. 博主诊断（核心）

**一个函数，两种目标模式，维度自适应**：

| 目标 | 维度 | 意义 |
|---|---|---|
| **诊断别人**（对标博主） | 硬实力 + **软实力(对路/可学/可蒸馏)** + 合规 | 值不值得对标 |
| **诊断自己**（我的账号） | 硬实力 + 合规（+ 短改进提示） | 我的号现状 + 有没有踩线 |

> 共享内核 = **硬实力 + 合规**（两种都跑）；**软实力只在诊断别人时出现**——"对路/可学"是"我能不能学 TA"，对自己无意义。
> 诊断别人时**不挂"我的账号"**：尺子是"你想对标什么"（对标意图），不是用户现状（现状乱、差距大是正常的，那是"找差距"的事）。

产出：**三区决策卡**，每维度 **0–100 + 理由**，分必须**有据**（能算的算，算不了才让模型判，每分附依据/引证）。

### 2.1 硬实力 —— 这号本身牛不牛（客观可算，行业指标背书）

| 维度 | 头牌指标 | 行业依据/阈值 | 建议算法（阈值后台可调） |
|---|---|---|---|
| **体量** | 粉丝量级 | KOL/KOC 分层 | `clamp(25*log10(粉丝+1),0,100)`；1k→75、1w→100；给档位标签 |
| **内容穿透** | 赞藏比 =(赞+藏)/粉 | ≥1.5 优质 | 账号总赞藏÷粉丝；≥3→100、1.5→75、<0.5→低；辅看"收藏/赞=干货度" |
| **爆款力** | 爆文率 + CES互动中位数 | 爆文=互动≥自己均值×3；中位>均值才真实 | 近N篇"互动≥自身中位×3"占比；CES互动=`赞×1+藏×1+评×4` 取中位数 |
| **更新活跃** | 频率+规律+活跃跨度 | 周3–7；活跃≥180天加成；忌刷屏 | 近90天频率 + 距最近一篇天数 + 间隔规律；过高频反扣 |
| **垂直度** | 主题集中度 | 60–80%；定位摇摆是大忌 | 模型看近N篇标题+正文是否聚焦一个赛道 |

> CES 权重来自小红书官方思路（`赞1 藏1 评4 转4 关8`）；采集后能拿全 赞/藏/评，CES 完整、抗"一篇爆款"干扰。

### 2.2 软实力 —— 我能不能学 TA（仅诊断别人，模型判，对标意图驱动）

**可学**（判据清单，多篇方法论收敛）：
- ✅ 可复制：选题**需求驱动**（非热点/标题党）；框架**模板化**；**不依赖个人IP/认证身份/特殊资源**；原生分享非硬广。
- ❌ 难复制：强个人IP、认证身份、特殊资源、纯热点、**AI模板痕迹重**（句式工整/三段式/emoji规律/多号同款）。
- 最狠一句：**爆款是因"选题匹配人群"（可学）还是"标题刺激/资源"（难复制）？**

**可蒸馏**：代表作篇数（算）+ 内容清晰度（模型）。

**对路 —— 怎么判（不靠拍脑袋，三面法）**：
1. **解析对标意图 → 三面**（模型把大白话拆开；用户可选微调）：
   - **题材**（什么赛道，如 香港保险/储蓄理财）
   - **形态**（专业硬核科普 / 图文为主，≠段子/vlog/纯种草）
   - **调性目标**（涨粉做专业IP，≠纯销售获客/卖课）
2. **从他真实笔记刻画同样三面**（采集到的标题+正文+笔记类型归纳）。
3. **逐面打匹配 + 必须引证**：题材匹配?(证据:近6篇全是港险科普) 形态匹配?(硬核科普图文=你要的专业) 调性匹配?(若每篇引导私信报价→偏销售获客，和"涨粉"部分不符⚠️) → 聚合成对路分，**每一面指着具体笔记说话**。

### 2.3 合规 —— 干不干净 / 学了安不安全（带否决权）

用 §3 引擎扫目标号近期笔记 → 合规分 + 分级风险点。语义：一个号哪怕硬高软高，靠**封号级违规打法**（保险"保证收益"、医美绝对功效、站外导流）做起来 →"学它=学封号"，结论降级，提示"选题结构能学、违规话术别碰"。**保险/医疗/金融这类资质赛道几乎必现且最关键。**

### 2.4 综合结论（诊断别人，三区联动，合规有否决权）

| 硬 | 软 | 合规 | 结论 |
|---|---|---|---|
| 高 | 高 | 干净 | **强烈推荐对标** |
| 高 | 高 | **封号级违规** | ⚠️ 慎：只学选题结构 |
| 高 | 低 | — | 它牛但不对你路 |
| 低 | 高 | — | 对路但天花板低（作补充） |
| 低 | 低 | — | 不建议 |

（诊断自己：硬实力档位 + 合规风险清单 + 改进提示，不出对标结论。）

### 2.5 决策卡 UI
三区竖排（诊断自己时只显硬实力+合规）。每条：维度 + 分 + 进度条 + 一句理由 + 证据/依据。底部：综合结论象限 + 操作（加入对标库已在库则显"去采集/蒸馏"）。

## 3. 合规共享引擎

### 3.1 现状
已有 `app/compliance/`：`wordlists.py`（极限词/医疗功效/诱导营销/竞品外链/基线）+ `scan.py`，只接了创作输出。本次升级为共享引擎 + 多处复用。

### 3.2 引擎升级：两层 + 品类感知 + 结构化
- **L1 词库/正则**：极限词、保证承诺、导流联系方式、诱导营销、**谐音/符号变体**；**新增** `金融保险·保证收益`、`品类红线词`。
- **L2 语义判**（模型）：隐性广告/虚假种草、**虚假人设**（虚构财富/学历/收入/情感，公约2.0）、夸大功效、**无资质荐险/具体投资建议**、品类越界、标题党。
- **品类感知**：传入品类（保险/金融/医疗/医美/美妆/教育/通用）套对应红线。
- **结构化输出**：每条 `{类别, 命中文本, 分级, 官方依据, 改写建议}` + 合规分（越高越干净）。**分级**：封号级 / 限流级 / 提示级。

### 3.3 复用三处
| 用在哪 | 喂什么 | 产出 |
|---|---|---|
| **博主诊断（他人/自己）** | 目标号近期笔记 | 合规维度 / 自己号的风险清单+整改建议 |
| **蒸馏skill输出文章** | 生成草稿 | 输出闸门（已存在→按新结构强化）：封号级拦 + 给改写 |

### 3.4 闭环
**诊断时挡掉"脏打法"的号 → 不把违规话术蒸进 Skill → 输出才不踩线。** 不在诊断端拦，违规会被蒸进 Skill 最后从用户嘴里说出来。

## 4. 数据来源与诚实天花板

诊断**前置一次采集**（已采 → 增量；未采 → 默认20条），所以诊断**一律基于真实笔记**：
- **能拿到（采集后）**：每篇 赞/藏/评（→ 完整 CES、爆文率、赞藏比）、正文/转写（→ 垂直度、可学性、对路三面）、发布时间（→ 更新活跃）、粉丝/笔记总数/总赞藏。
- **永远拿不到（必须诚实标注，不可假装）**：**真·互动率（无阅读量）、阅读中位数、粉丝画像、历史涨粉速**。
→ 互动质量用 **赞藏比 + 爆文率 + CES中位数**（均不依赖阅读量）替代。**不再有"轻量估算"两档**——诊断既然采过，就都是真实数据。

## 5. 代码改造点

- **删**：`benchmark_discovery/{flow,recall,vetting,querygen}` + discovery 端点/任务；前端泛搜索/找相似/独立"诊断我的"页。
- **改 `app/compliance/`**：`wordlists.py` 补金融保险/品类红线 + 分级；新增 `engine.py`（L1+L2+品类，结构化）；`scan.py` 改走新引擎。
- **演进 `account_audit` → 博主诊断**：保留 self/benchmark 两 kind；接入新的 硬实力算法 + 软实力(benchmark) + 合规适配；确保诊断前"≥20条笔记"（增量/默认采20）。
- **硬实力纯函数**（赞藏比/爆文率/CES/更新/垂直）→ 单独模块，重单测。
- **config + admin**：硬实力阈值、爆文倍数、采样N、默认采集条数(20)、合规分级权重、品类词库 → `config.py` + `admin/config_registry`。
- **前端**：精确搜索（预览+加库）；新导航子模块 **对标分析**（博主诊断 + 找差距占位）；三区决策卡；创作输出沿用强化后的闸门。

## 6. 实现分期（建议）

1. **P1 砍泛搜索**：删前端页 + 后端 discovery（路由快照确认其余不变）+ 删独立"诊断我的"页。
2. **P2 合规引擎升级**：wordlists 补全 + engine(L1+L2+品类) + 结构化 + 单测。
3. **P3 硬实力内核**（纯函数，重单测）+ 软实力 prompt(对路三面/可学/可蒸馏) + 合规适配。
4. **P4 博主诊断编排**（self/benchmark、采集前置 ≥20条）+ API。
5. **P5 前端**：对标分析子模块 + 三区决策卡 + 精确搜索改造；创作输出闸门改走新引擎。
6. **P6 验证 + 部署**：单测 + ruff + build + 真机跑香港保险一个号验证三区 + 诊断自己。

## 7. 本轮不做

- **找差距**（我 vs 对标）—— 归属"对标分析"模块，**下一轮单独设计**；博主诊断这里不碰差距比较。
- 找相似 / 泛搜索任何自动发现 —— 砍。
