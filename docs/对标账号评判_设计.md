# 对标账号评判 + 合规共享引擎 —— 方案设计

> 状态：**设计稿**（2026-06，待实现）。本文件取代 [[找对标_泛搜索流程设计]]（泛搜索已决定砍掉，原因见下）。
> 关联：[[采集模块设计]]、[[阶段B_蒸馏选材改造设计]]、[[商业模式与定位]]。
> 一句话：**砍掉冷启动泛搜索**，只留**精确搜索（名字/ID）**找到一个具体博主，核心做**「这个号到底值不值得对标学习」的评判**——从 **硬实力（牛不牛）× 软实力（对不对你路）× 合规（干不干净）** 三个维度给分、给理由；同一套**合规引擎**还复用到「诊断我的账号」和「蒸馏 Skill 输出文章」。

---

## 0. 为什么推翻泛搜索

泛搜索（领域 → AI 推细分领域 → 关键词广召回 → 核验打分 → 候选）实测给不出实质价值，根因是**结构性**的，不是调参能救的：

1. **采料口错**：关键词搜"账号名"只命中**把词写进昵称/简介的小号**（从业者/销售），真正做得好的大号要么人设更宽、要么主动**藏限流词**（黑话/截图/私信），关键词根本搜不到。香港保险实测：候选粉丝 2～1573，全是on-the-nose小号。
2. **相关 ≠ 值得对标**：相关性评委奖励"够纯"，但越纯往往越是小销售号；越值得学的大创作者往往越"不纯"。
3. **平台能力缺失**：TikHub 对小红书**只有扁平关键词搜索 + 单号抓取**，没有"推荐引擎 / 同类账号 / 类目榜单"图谱接口。大号是被小红书**推荐 + 人滚雪球浏览**找出来的，不是被"搜"出来的——这能力我们零接入。
4. **找相似也跑偏**：基于种子博主的"找相似"会让 AI 凭空造抽象细分领域（实测推出"抖音带货话术""小红书美妆定位"），没吃种子真实内容。

**结论**：发现（找）这件事，**用户 + 小红书 App 本身远强于我们**，别假装能自动找。我们的不对称优势在 **判 / 蒸**——所以重心放"评判一个号值不值得学"。

### 砍掉的东西
- 前端：泛搜索页、找相似子 tab、候选/种子/已选三列工作台。
- 后端：`benchmark_discovery` 的 `flow / recall / vetting / querygen`、`/benchmark/discovery/*` 全部端点、discovery 会话相关任务。
- 保留并复用：搜索（`blogger_distillation/search`）、单号资料/笔记抓取、`evaluate_one` 的骨架（它本就吃 intent）。

## 1. 精确搜索（唯一的"找"入口）

- 支持**按名字**或**按小红书号/ID**搜索（两条路都通；落地时确认 search 接口对 ID 直查的支持）。
- 不做 URL 粘贴（用户反馈太麻烦）。
- 搜索结果点一个 → 进入「账号评判」。

---

## 2. 账号评判（核心）

输入：① 选中的博主（拉主页 + 近 N 篇笔记）；② 一句**对标意图**（"你想对标什么样的博主 / 想学什么"，例：想学把香港保险讲专业又涨粉）。
**不挂"我的账号"**——评判的尺子是"用户想对标什么"，不是用户现状（现状很乱、跟对标差距大是正常的，那是后续"找差距"的事，不该影响"值不值得学"的判断）。

产出：一张**三区决策卡**（硬实力 / 软实力 / 合规），每维度 **0–100 分 + 理由**，分必须**有据**（能算的算，算不了才让模型判，每分附依据）。

### 2.1 硬实力 —— 这号本身牛不牛（客观可算，行业指标背书）

| 维度 | 头牌指标 | 行业依据/阈值 | 建议算法（阈值后台可调） |
|---|---|---|---|
| **体量** | 粉丝量级 | KOL/KOC 分层 | `clamp(25*log10(粉丝+1),0,100)`；1k→75、1w→100；并给档位标签（素人<1k / KOC 1k–1w / 腰部 1w–10w / 头部>10w） |
| **内容穿透** | 赞藏比 =(赞+藏)/粉 | ≥1.5 优质号 | 账号总赞藏÷粉丝；≥3→100、1.5→75、<0.5→低；辅助看"收藏/赞=干货度" |
| **爆款力** | 爆文率 + CES互动中位数 | 爆文=互动≥自己均值×3；中位数>均值才算真实水位 | 近N篇里"互动≥自身中位数×3"的占比 → 爆文率；CES互动=`赞×1+藏×1+评×4` 取**中位数** |
| **更新活跃** | 频率+规律+活跃跨度 | 周3–7篇；活跃≥180天加成；忌刷屏掉权重 | 近90天发文频率 + 距最近一篇天数 + 间隔规律性；过高频(刷屏)反扣 |
| **垂直度** | 主题集中度 | 60–80%；定位摇摆是大忌 | 模型看近N篇标题/话题是否聚焦一个赛道 → 0–100 |

> **CES 权重来自小红书官方思路**：`赞×1 + 收藏×1 + 评论×4 + 转发×4 + 关注×8`。我们能拿到 赞/藏/评，按 `赞×1+藏×1+评×4` 近似，比纯点赞专业且抗"一篇爆款"干扰。

### 2.2 软实力 —— 我能不能用（对标意图驱动，模型判）

| 维度 | 衡量 | 怎么判 |
|---|---|---|
| **对路** | 内容方向 vs 你想学的 | 模型 × 对标意图（不是关键词纯度，是"是不是你想成为的那种"） |
| **可学** | 真人方法论 vs 纯销售/硬广 | 模型按下面这套**有依据的清单**判 |
| **可蒸馏** | 能不能蒸出方法论 | 代表作篇数（算）+ 内容清晰度（模型） |

**可学性判据清单**（多篇行业方法论收敛）：
- ✅ 可复制：选题**需求驱动**（非热点/标题党）；内容**框架模板化**（如"痛点→对比→避坑→推荐"）；**不依赖个人IP/认证身份/特殊资源**；原生分享非硬广。
- ❌ 难复制/慎学：强个人IP、认证身份（医生/律师）、特殊资源、纯热点追踪、**AI模板痕迹重**（句式工整/三段式/emoji规律/多号同款）。
- 最狠一句：**爆款是因为"选题匹配人群"（可学），还是"标题刺激/资源"（难复制）？**

### 2.3 合规 —— 干不干净 / 学了安不安全（第三大维度，带否决权）

用 §3 的合规引擎扫该博主近期笔记，产出**合规分 + 风险点**。语义：一个号哪怕硬高软高，只要靠**封号级违规打法**（如保险"保证收益"、医美绝对功效、站外导流）做起来，**学它=学封号**——结论直接降级，提示"它的选题结构能学，但**别学它的违规话术**"。

> 对**保险/医疗/金融**这类**资质强相关赛道，合规维度几乎必现且最关键**——它正好解释了大号为什么用"保护性措施"。

### 2.4 综合结论（三区联动，不是简单加权）

三区**各出一个分**，结论按规则给（合规有否决权）：

| 硬 | 软 | 合规 | 结论 |
|---|---|---|---|
| 高 | 高 | 干净 | **强烈推荐对标** |
| 高 | 高 | **封号级违规** | ⚠️ 慎：只学选题结构，违规话术别碰 |
| 高 | 低 | — | 它牛但不对你路/搬不动 |
| 低 | 高 | — | 对路但它本身一般，学了天花板低（只作补充） |
| 低 | 低 | — | 不建议对标 |

合格（推荐/可补充）→ 一键 **加入对标库 → 采集 → 蒸馏**（沿用现成）。

### 2.5 决策卡 UI
三区竖排：硬实力（5 条带分条）/ 软实力（3 条）/ 合规（分 + 分级风险清单）。每条：维度名 + 分 + 进度条 + 一句理由 + 数据来源标注（"轻量估算 / 采集后精确"）。底部：综合结论象限 + 操作按钮。

## 3. 合规共享引擎（一个引擎，三处复用）

### 3.1 现状
已有 `app/compliance/`：`wordlists.py`（类目：极限词 / 医疗功效 / 诱导营销 / 竞品外链 / 基线）+ `scan.py`（`scan_creation_output`），但**只接了创作输出一条线**（`xhs_creation/agent` 的 prompt + sensor）。本次**升级为共享引擎 + 接到另外两个面**。

### 3.2 引擎升级：两层 + 品类感知 + 结构化输出

- **L1 词库/正则**（快、准、便宜）：极限词、保证承诺、导流联系方式（微信/V/电话/外链）、诱导营销、**谐音/符号变体**。在现有类目上**新增**：
  - `金融保险·保证收益`（保本/稳赚/承诺年化/零风险/稳赚不赔）
  - `品类红线词`（按品类挂）
- **L2 语义判**（模型，抓词库抓不到的）：隐性广告/虚假种草、**虚假人设**（虚构财富/学历/收入/情感，公约2.0）、夸大功效、**无资质荐险/给具体投资建议**、品类越界、标题党猎奇。
- **品类感知**：传入 `品类`（保险/金融/医疗/医美/美妆/教育/通用…），套对应红线规则集。
- **结构化输出**：每条 `{类别, 命中文本, 分级, 官方依据, 改写建议}` + **合规分**（100 − 加权扣分，越高越干净）。
- **分级**：`封号级`（导流/保证收益/无资质荐荐/绝对功效）、`限流级`（极限词/夸大/标题党）、`提示级`（轻微/边界）。

### 3.3 三个适配器（同引擎，不同入口/产出）

| 适配器 | 喂什么 | 产出 | 复用现状 |
|---|---|---|---|
| **① 评判对标博主** | 它的近期笔记 | 合规维度（§2.3）：干不干净 / 学了安不安全 | 新接 |
| **② 诊断我的账号** | 我的笔记 | 风险笔记清单 + 违规词/句 + 整改建议 + 合规分 | 接入 `account_audit` |
| **③ 蒸馏skill输出** | 生成的草稿 | 输出闸门：封号级拦截 + 给改写 | **已存在**，按新结构强化 |

### 3.4 闭环（合规为什么要串三处）
**评判时挡掉"脏打法"的号 → 不把违规话术蒸进 Skill → 输出文章才不踩线。** 三处是一条链：不在评判端拦，违规会被蒸进 Skill，最后从用户嘴里说出来。

## 4. 数据来源与诚实天花板

**TikHub 一次轻量抓取能拿到**：粉丝数、笔记总数、账号总赞藏、近 N 篇（标题、点赞、发布时间、笔记类型、xsec_token）。
**采集后更精确**：每篇 收藏/评论（→ 完整 CES）、正文/转写（→ 垂直度、可学性更准）。
**永远拿不到**（必须诚实标注、不可假装）：**真·互动率（无阅读量）、阅读中位数、粉丝画像、历史涨粉速**。
→ 因此互动质量用**赞藏比 + 爆文率 + CES中位数**这套**不依赖阅读量**的替代；卡片标注"轻量估算 / 采集后精确"。

## 5. 代码改造点（实现时）

- **删**：`app/benchmark_discovery/{flow,recall,vetting,querygen}.py` 及 discovery 端点/会话任务；前端泛搜索 + 找相似页。
- **改 `app/compliance/`**：`wordlists.py` 补金融保险/品类红线 + 分级；新增 `engine.py`（L1+L2+品类，结构化输出）；`scan.py` 保留并改走新引擎。
- **新增评判内核**：`benchmark_discovery`（或新包）下的 `appraisal`——硬实力算法（赞藏比/爆文率/CES/更新/垂直）+ 软实力 prompt（对路/可学/可蒸馏）+ 合规适配器，产出三区决策卡。
- **复用**：`evaluate_one`（改造成三区输出）、`account_audit`（接合规适配器②）、`xhs_creation/agent/sensors`（适配器③改走新引擎）。
- **config + admin**：硬实力阈值、爆文率倍数、垂直度采样 N、合规分级权重、品类词库 → 入 `config.py` + `admin/config_registry`（后台可调）。
- **前端**：精确搜索页 + 三区决策卡 + 我的账号合规体检展示。

## 6. 实现分期（建议）

1. **P1 砍泛搜索**：删前端页 + 后端 discovery（路由快照确认其余不变）。
2. **P2 合规引擎升级**：wordlists 补全 + engine（L1+L2+品类）+ 结构化 + 单测。
3. **P3 评判内核**：硬实力算法（纯函数，重单测）+ 软实力 prompt + 合规适配器①。
4. **P4 评判 API + 三区决策卡前端**。
5. **P5 接入②③**：account_audit 合规体检、创作输出闸门改走新引擎。
6. **P6 验证 + 部署**：单测 + ruff + build + 真机跑香港保险一个号验证三区。

## 7. 本轮不做

- **找差距**（我 vs 对标，account-vs-blogger）——单独设计文档，下一轮；评判这里**不碰我的账号**。
- 找相似（从种子扩同类）——砍。
- 泛搜索任何形式的自动发现——砍。
