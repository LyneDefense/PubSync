# 博主画像模块 · 全景设计（模块总纲）

> 本文是**模块级总纲**,定位"博主画像是什么、有几层、对用户什么价值、在系统里什么作用"。
> 其下的实现级文档:[[蒸馏产出重定义_方案]](定义 ④⑤ 层产出什么)、[[Skill能力包升级_方案设计]](④⑤ 怎么被创作消费)、[[蒸馏证据供给优化_方案设计]](已实施,让蒸馏看得全)。

## 一、模块定位:系统的知识中枢

博主画像**不是一个孤立功能,是整个系统的知识核心 / 单一事实源**。所有下游(创作/选题/诊断/推荐/看板)都不直接碰原始笔记,都从画像取所需。一句话:

> **采集喂原料 → 画像组织成立体理解 → 下游各取所需。** 画像做厚做准,整个系统一起受益。

## 二、"立体"的含义:五层,回答五个问题

| 层 | 回答 | 提供的数据 | 数据源 | 基数 | 产线 |
|---|---|---|---|---|---|
| **① 身份** | 她是谁 | 昵称·粉丝·总笔记·赛道·简介·人设 | 主页元数据(1 次) | 唯一·可刷新 | 账号事实 |
| **② 数据** | 表现如何 | 均赞/藏/评/浏览·藏赞比·互动率·图文vs视频分布 | 全量列表(~$0.16) | 唯一·可刷新 | 账号事实 |
| **③ 轨迹** | 怎么走到今天 | 爆文时间线·低谷/爆发/平稳期·**转折点+爆文归因**·近期趋势 | 全量列表 create_time 时序 | 唯一·可刷新 | 账号事实 |
| **④ 认知** | 怎么想 | 核心信念·观点张力·价值立场·人设内核 | 快照蒸馏(LLM) | 随创作画像·可多个 | 创作画像 |
| **⑤ 方法** | 怎么做内容 | 分车道写法·选题策略·对比示例 | 快照蒸馏(LLM) | 可多个 | 创作画像 |

**两条产线**:①②③ = **账号事实**(全量算、唯一、便宜、来自 TikHub);④⑤ = **创作画像**(样本蒸、可多个、来自 LLM)。

## 三、逐层详解

### ① 身份层 — 她是谁
基本档案 + 人设。数据源:博主主页元数据(`get_user_info`,1 次调用)。现状:`BloggerProfile` 已覆盖大部分(昵称/主页/赛道/note_total 等)。

### ② 数据层 — 表现如何
账号级互动统计:均赞/藏/评/**浏览**、藏赞比、**互动率**(赞藏评÷浏览)、图文 vs 视频占比及各自均赞。**必须从全量算**,不能从快照子集(否则失真,见轨迹层同一 bug)。

### ③ 轨迹层 — 怎么走到今天(含爆文归因)
本层是画像"立体"的关键,分三件事:

**(1) 爆文时间线**:用全量列表的 `create_time` + 互动画一条"内容表现曲线"(实测可行:301 篇、含 create_time/赞/藏/评/浏览,~$0.16)。业内公认"爆文曲线 ≈ 涨粉曲线",且官方涨粉曲线拿不到(见 [[蒸馏产出重定义_方案]] 第十节),故用内容表现近似。

**(2) 阶段识别**:低谷期(高产低互动)/ 爆发期(互动量级跃升)/ 平稳期。**判据用"相对自身基线的跃升 + 量级台阶",不用绝对值**(早期互动低可能是粉丝基数小,非内容差)。要能识别多种形态(成长型/腰部型/衰退型),不假设人人"从不火到火"。

**(3) 爆文归因**(明确能力,但定位为"有据的假设",非因果证明):
- **A. 转折点变量对比**:爆发前后**可观测地**变了什么——选题/形式(图文↔口播)/标题公式/封面/发布节奏。硬数据。
- **B. 爆文共性**:爆文组 vs 普通组对比,提炼共同因子。**必须拿非爆文当对照,防幸存者偏差**。硬统计。
- **C. LLM 因果假设**:把 A+B 喂 LLM,生成"她可能靠 X 起飞"的推测,**标注为假设**。软。
- **方法论**:归因靠"**对比**"(爆vs不爆、转折前vs后),不是看曲线本身;曲线只告诉"何时爆"。
- **三个坑**:① 相关≠因果(可能是平台推流/蹭热点)② 幸存者偏差(必须对照非爆文)③ 看不到的外部变量(推荐权重/热点/大V转发/账号互推);单点爆款可能纯运气,只追"多篇爆文的共性"(可复制模式)。
- **产出即桥**:归因产出("她成功时的打法")= ③轨迹 → ⑤方法 的桥——识别爆发期 → 蒸那批笔记 → 得"成功转折方法论"。

### ④ 认知层 — 怎么想
核心信念/观点张力/价值立场/人设内核。从快照蒸馏。主要服务**选题**(观点张力=选题角度金矿),创作只借"薄人设声音"。现状:混在 `BloggerSkill`,待按 [[蒸馏产出重定义_方案]] 拆出。

### ⑤ 方法层 — 怎么做内容
分车道(图文/口播/非口播)写法 + 选题策略 + 对比示例。从快照蒸馏,**可多个画像**(爆款/垂类/近期)。现状:即当前 `BloggerSkill` 的主体。

## 四、模块功能(三个动作)

1. **建 / 刷新画像**:拉全量列表 + 主页 → 更新 ①②③(便宜、可常刷)。
2. **蒸馏创作画像**:选快照 → 产 ④⑤(LLM,同一博主可蒸多个侧面)。
3. **展示**:五层各有呈现——档案卡 / 数据面板 / **成长曲线** / 认知卡 / 创作手册。

## 五、对用户的帮助(三个递进场景)

1. **"这博主值不值得我学?"** → 看 ②③:她真火吗、在涨还是在跌 → **对标决策**,避免对标衰退号/数据虚高号。
2. **"她凭什么火起来的?"** → 看 ③ 的转折点+归因:起飞前后变了什么、爆文共性 → **成长复盘**。
3. **"我怎么写出她那味儿?"** → 看 ④⑤:怎么想、怎么写 → 喂创作/选题 → **创作借鉴**。

价值链:**从「值不值得学」→「凭什么火」→「我怎么学」,一个画像全包。**

## 六、在系统里的作用(中枢 + 上下游)

- **上游喂它**:采集(全量列表→①②③;快照→④⑤蒸馏)、视觉/ASR(内容理解→喂蒸馏)。
- **下游全靠它**:
  - 创作 skill ← ⑤ + 薄人设
  - 选题器 ← ④ + ③ 的热点/趋势
  - 对标诊断(appraisal)← ②③
  - 对标推荐/筛选(benchmark_discovery)← ②③ 帮用户挑该学谁
  - 效果看板 ← 用户自己 vs 对标画像 对比

## 七、与现有实现的映射(哪些已有 / 哪些要建)

| 层 | 现状 | 要做 |
|---|---|---|
| ① 身份 | `BloggerProfile` 已覆盖大部分 | 基本够,补人设可选 |
| ② 数据 | `analysis.py` 有统计,但**从快照子集算(bug)** | 移到**全量**算 |
| ③ 轨迹 | **无** | **全新建**:全量列表时序 + 阶段识别 + 爆文归因 |
| ④ 认知 | 混在 `BloggerSkill`/蒸馏 core | 按 [[蒸馏产出重定义_方案]] 拆出 |
| ⑤ 方法 | 即当前 `BloggerSkill` 主体 | 收窄成"创作方法",按车道投影 |

## 八、落地次序(建议)

1. **拆产线 + 修数据 bug**:账号事实统计(②)从蒸馏移出,改全量算(地基,见 [[蒸馏产出重定义_方案]] 落地三件事之一)。
2. **轨迹层 MVP**:轻量"只翻列表"通路(加 `create_time` 解析)→ 落库时序 → 阶段识别 + 曲线。成本已验证 ~$0.16/博主。
3. **爆文归因**:在轨迹层上做对比组(爆vs普通)+ 共性 + LLM 假设。
4. **画像正名 + 多画像**:④⑤ 收窄、命名、松开单 active。
5. 展示层:五层可视化。

## 九、边界

- 归因是"有据的假设",不承诺因果证明。
- 官方涨粉曲线拿不到,用内容表现曲线近似(诚实标注)。
- 大号(>600 篇)只取最近段,`reached_end` 标记是否翻到第一篇。
- 不做标准 Agent Skill 导出包(已决定暂不需要)。
