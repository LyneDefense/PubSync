# 合规检测 · 架构重构方案

> 目标：把现在「单层无上下文子串词库 + LLM 加法器 + 全量平铺」的合规检测，重构成 **precision-first、赛道感知、可扩展、分级输出** 的架构。
> 两根正交的轴：**级联引擎（怎么判）× 规则包注册表（判什么）+ LLM 长尾兜底**。

---

## 0. 现状与问题（为什么要重构）

现有 `app/compliance/`：
- `wordlists.py`：~150 词 / 8 类，其中 7 类是**通用类目**（对所有账号生效），只有 1 个**行业包**（金融 `FINANCE_GUARANTEE`）。
- `engine.py::scan_l1`：**裸子串匹配** `词 in 文本`，扫每篇笔记标题+正文，按 (类别,词) 去重。
- `engine.py::scan_l2`：LLM 在 L1 之上**额外**挑语义问题（加法）。
- `scan.py::scan_creation_output`：创作闸门用的同款子串扫描。
- 输出：`hits[]` 全量平铺，前端逐条列。

**四个结构性硬伤**（诊断一个真实账号 30–40 篇 → 10+ 命中，一大半误报）：
1. **无上下文子串**：`最好`命中「还是最好早点睡」、`抖音`命中「在抖音刷到」、`美白`命中护肤卖点描述。
2. **医疗当通用类**：任何账号（含旅游/数码）都被拿「美白/祛痘」扫，且判**封号级**——而这些其实是**合法化妆品宣称**。
3. **只有一个行业**：金融接了，其余赛道（母婴/教育/食品/房产/招商…）完全没有专项规则。
4. **LLM 是加法器 + 全量平铺**：L2 只增不减；提示级与封号级同权铺开，数量吓人。

## 1. 目标架构总览（两轴 + 三层引擎）

```
账号(niche/tags/标题)
   │
   ▼
[赛道识别 vertical.py] ──► 激活规则包: 通用包 + 命中的行业包(可多个) + 平台包
   │
   ▼   （引擎轴，与赛道无关，三层级联）
┌─────────────────────────────────────────────┐
│ 一层 matcher.py   确定性规则                 │  便宜、可审计
│   词边界 + 白名单 + 上下文规则 → 打置信度    │  certain 直接判
├─────────────────────────────────────────────┤
│ 二层 adjudicator.py  LLM 裁判（只处理少数） │  policy-as-prompt
│   裁 weak/模糊命中 + 长尾赛道 + 语义补漏     │  减法为主
├─────────────────────────────────────────────┤
│ 三层 grade.py   归并 / 分级 / 占比 / 算分    │  违规 vs 提示两档
└─────────────────────────────────────────────┘
   │
   ▼
ScanResult { violations[], advisories[], score, verdict, coverage, verticals }
```

- **引擎轴**：级联。绝大多数由一层确定；只有模糊的、长尾赛道的交给二层 LLM。LLM 是「疑难案件的最后一道」，不是前门。
- **规则轴**：规则包注册表。通用包所有人生效；行业包按赛道激活（**不激活就完全不扫，从根上消灭跨赛道误报**）；平台包按平台。
- 加新赛道 = 丢一个 pack，引擎零改动。

## 2. 核心数据结构（`app/compliance/types.py`，新增）

```python
class Severity(str, Enum):        # 对齐平台阶梯处罚
    BAN = "封号级"; THROTTLE = "限流级"; NOTICE = "提示级"

class Confidence(str, Enum):
    CERTAIN = "certain"           # 直接定性(如 保本/加微信)
    PROBABLE = "probable"         # 大概率(交 LLM 裁可选)
    WEAK = "weak"                 # 易误报,必须 LLM 裁或降级为提示

@dataclass
class Rule:
    id: str
    words: list[str] = ()             # 字面词(literal)
    patterns: list[str] = ()          # 正则(含上下文/邻近)
    allow_words: list[str] = ()       # 白名单:命中词落在这些更长词/语境里则抑制(抗皱⊃皱、办公⊃公安)
    allow_context: list[str] = ()     # 白名单语境正则(如 极限词前有"还是/最"→口语,抑制)
    require_context: list[str] = ()   # 必须邻近才算(平台名 only if 邻近 去/上/搜/私域/加)
    severity: Severity = Severity.NOTICE
    confidence: Confidence = Confidence.CERTAIN
    basis: str = ""                   # 法规/平台条文依据
    hint: str = ""                    # 改法

@dataclass
class RulePack:
    id: str                           # "common" / "cosmetics" / "finance" ...
    label: str                        # 展示名
    scope: str                        # "universal" | "vertical" | "platform"
    activate_verticals: list[str] = ()# vertical 命中其一则激活(universal/platform 忽略)
    platforms: list[str] = ()         # platform 包限定平台
    rules: list[Rule] = ()
    version: str = ""                 # 法规时效(如 "2026-02 三品一械")

@dataclass
class Hit:                            # 单条命中(一层产出的候选/最终确认)
    pack_id: str; rule_id: str; category: str
    matched: str; quote: str; field: str
    severity: Severity; confidence: Confidence
    basis: str; hint: str
    layer: str                        # "rule" | "llm"
    status: str = "candidate"         # candidate | confirmed | dismissed

@dataclass
class ScanResult:
    score: int; verdict: str
    violations: list[dict]            # 封号/限流(需处置)——按规则归并后
    advisories: list[dict]            # 提示级(优化建议,不计入违规数)
    coverage: dict                    # {rule_id: {"hit_notes": n, "total_notes": m}}
    verticals: list[str]             # 命中的赛道(展示"按 X 赛道检测")
    by_severity: dict; has_ban: bool
```

## 3. 模块拆分（后端）

```
app/compliance/
  __init__.py          # 对外: scan_account(...) / scan_creation(...) / ScanResult
  types.py             # ← 第 2 节的 dataclass/enum
  vertical.py          # 赛道识别: 关键词映射 + LLM 兜底
  registry.py          # 规则包注册 + 激活(通用+行业+平台)
  matcher.py           # 一层: 归一化 + 词边界 + 白名单 + 上下文 + 置信度 → 候选命中
  adjudicator.py       # 二层: LLM 裁判(裁 weak + 长尾 policy-as-prompt + 语义补漏)
  grade.py             # 三层: 归并/分级/占比/算分 → ScanResult
  packs/
    common.py          # 通用包: 广告法绝对化/导流/诱导/基线违禁
    platform.py        # 平台包: xhs/douyin/wechat(禁推类目/导流差异)
    cosmetics.py       # 美妆护肤(功效"合法vs禁")
    medical.py         # 医疗健康(三品一械/疗效/诊疗)
    food_health.py     # 食品保健(疗效/零添加)
    maternal.py        # 母婴(母乳宣称/医用级/焦虑营销)
    finance.py         # 金融保险(迁移现有 FINANCE_GUARANTEE)
    education.py       # 教育培训(升学/通过承诺)
    fitness.py         # 减肥健身(快速瘦身/燃脂)
    recruit.py         # 招商加盟(暴利/躺赚/传销)
    __init__.py        # 汇总 ALL_PACKS
```

- `engine.py` / `wordlists.py` / `scan.py` **保留一版做迁移垫片**，切换完再删；或直接改 `__init__.py` 的对外函数签名，内部指向新链路。
- **jieba 不引入**（当前非依赖，体积大）：一层用**轻量词边界启发式**（命中词前后字符的类别：汉字/标点/空白 + 白名单语境正则）就能干掉绝大多数子串误报；若后续证明不够，再评估 jieba。

## 4. 一层 · 确定性匹配（`matcher.py`）

1. **归一化**：全角→半角、简繁统一、去零宽/分隔符（保留 position mapping，命中片段能回原文）。
2. **匹配**：对每个激活包的每条 `Rule`：
   - `words` 子串命中 → 过**词边界 + `allow_words`（更长词吸收，如 抗皱⊃皱、办公安排⊃公安）+ `allow_context`（口语语境抑制，如 "还是最好"）**；
   - `require_context` 存在时，只有命中词**邻近**要求语境才算（平台名 only if 邻近 去/上/搜/买/私/加微）；
   - `patterns` 正则命中同样过白名单。
3. 每条候选 `Hit` 带 `confidence`；`certain` 直接 confirmed，`probable/weak` 标 candidate 待二层裁。
4. 产出 `list[Hit]`（候选集）+ 每条的命中笔记数（做覆盖率）。

> 这一层就能实现 **P1 止血**：赛道门控 + 词边界 + 白名单 + 上下文，误报洪水当场消掉，且全离线可测、零 LLM。

---

## 5. 二层 · LLM 裁判（`adjudicator.py`，policy-as-prompt）

只在 `use_llm=True`（诊断）时跑；候选为空则不调用。三件事：
1. **裁模糊命中**：把一层的 `weak/probable` 候选 + 命中原句 + 该规则 `basis` 条文喂给模型，输出每条 `keep/dismiss + 理由 (+ 可修正 severity)`。→ LLM 做**减法/证伪**。
2. **长尾赛道兜底**：账号赛道没有手写包时，注入「该赛道适用的广告法+平台规则」作为 policy，让模型直接判违规原句+条文+分级+改法。→ 广度靠 LLM。
3. **语义补漏（受限）**：只找词库抓不到的（隐性广告/虚假人设/夸大功效），**必须引用原文**，宁缺毋滥。
4. **失败/超时**：降级——`weak` 候选按配置「保留为提示级」或「丢弃」，`certain` 不受影响，绝不阻断。

---

## 6. 三层 · 归并分级算分（`grade.py`）

1. **归并**：同 (pack, rule) 的多条命中并成一条，带命中样例 + `coverage`（命中 X / 共 Y 篇）。
2. **两档**：`violations`（封号/限流，需处置） vs `advisories`（提示级，优化建议，**不计入违规数**）。
3. **算分**：只由 confirmed `violations` 决定，每类封顶（沿用 `_PER_CATEGORY_CAP`），提示级不扣或轻扣。
4. **verdict**：干净 / 边界 / 有限流风险 / 高危（含封号级）。

---

## 7. 赛道识别（`vertical.py`）

- 输入：`blogger.niche` + `tags` + 近期标题。
- **一级 关键词映射**（覆盖头部，零成本）：`{护肤/美妆/彩妆→cosmetics, 保险/理财/基金→finance, 育儿/母婴→maternal, 考研/雅思/公考→education, 减脂/健身→fitness, 养生/保健→food_health, 医美/医疗→medical, 加盟/创业→recruit, ...}`。
- **二级 LLM 兜底**：一级判不出 → 一次便宜调用，从**固定枚举**里归 1–2 个赛道 label。
- **多赛道**：返回 `list[str]`，激活多个行业包（美妆+好物 → cosmetics 生效，通用包兜底其余）。
- 通用包 always；平台包 by `platform`。
- 结果可缓存（`BloggerProfile` 加 `verticals_json` 列，或每次算——量小可不缓存）。

## 8. 两个调用方 · 操作点（precision/recall 分开）

| 调用方 | 场景 | 操作点 |
|---|---|---|
| `scan_account`（诊断别人/自己） | 判真实号 30–40 篇 | **精确优先**：默认只报 `certain + LLM-confirmed`；advisories 折叠 |
| `scan_creation`（创作闸门） | 发布前扫自己草稿 | **略偏召回**：连 `weak` 也提示（自己可改）；`creation_compliance_enabled` 沿用 |

通过参数 `operating_point(min_confidence, include_advisories, use_llm)` 区分，不共用死设定。

---

## 9. 前端改动（`AppraisalCard.vue` 合规区 + 创作合规展示）

- **违规区**：`violations` 按类别分组卡——类别 + 依据(条文) + **命中 X/Y 篇** + 改法 + 1–2 条样例，不再逐词平铺。
- **提示区**：`advisories` 折叠成「N 条优化提示」，点开才展开，不计入「违规」计数。
- **赛道标签**：顶部「按【美妆护肤 · 通用】规则检测」，让用户知道扫了哪些包。
- **误报反馈**（P3）：每条旁「这不算违规」→ 落白名单。

## 10. 配置 / 后台 / 自定义

- `config.py`：新增 `compliance_use_llm_adjudicator: bool`、`compliance_min_confidence`（诊断默认 certain）、`compliance_llm_timeout`（复用 appraisal 超时）。
- `admin/config_registry.py`：暴露上述开关 + 「按赛道启用/停用规则包」。
- 自定义扩展：把现有「一坨扩展词」升级为 **per-vertical 自定义规则**（admin 可给某赛道加词/加白名单）。

## 11. 兼容与迁移

- **`report_json.compliance` 结构升级**：新增 `violations/advisories/coverage/verticals`；**保留 `hits/score/grade/by_severity`**（`hits = violations + advisories` 并集），旧报告与旧前端不炸；前端同步改成读新字段。
- 旧 `wordlists.py` 词表 → 拆进 `packs/*`；`resolve_industry` → `vertical.py`。
- 现有 `compliance_scan` / `scan_creation_output` 签名保留为**薄适配层**，内部转调新 `scan_account/scan_creation`，逐步替换调用点（`appraisal/service.py`、创作 service）。

## 12. 分期落地

| 期 | 内容 | 价值 | 风险 |
|---|---|---|---|
| **P1 止血** | `types/registry/matcher/grade` + `vertical`(仅关键词映射) + `common/platform/finance/cosmetics/medical` 五包；**医疗从通用降为赛道包**；词边界+白名单+上下文；两档分级输出；前端读新字段 | 误报洪水当场消除、数量不再吓人、跨赛道不再乱扫 | 低（纯确定性、全离线可测、无 LLM 变更） |
| **P2 引擎级联** | `adjudicator`：LLM 从加法器改**裁判**（裁 weak + 语义补漏）；两调用方操作点；vertical 加 LLM 兜底 | 精度再上一档、召回补齐 | 中（LLM 成本；已并行、只裁候选子集可控） |
| **P3 全赛道 + 闭环** | 补 `food_health/maternal/education/fitness/recruit` 等头部包；长尾 policy-as-prompt；误报反馈→白名单；admin per-vertical 规则 | 覆盖全赛道、自我进化 | 中（规则整理工作量） |

## 13. 测试策略

- **规则包正/反例**：每条规则至少 1 正例(该报) + 1 反例(白名单，不该报)。硬指标：
  - `抗皱/减少皱纹` 不报、`祛皱/抗菌` 报（cosmetics）
  - 「还是最好早点睡」不报、「全网最好用」报（common 极限词）
  - 「在抖音刷到」不报、「去抖音搜同款」报（platform 导流上下文）
  - `保本/稳赚/加微信` 报（finance/common）
- **赛道识别单测**：护肤→cosmetics、保险→finance、旅游→无行业包。
- **grade 单测**：归并、覆盖率、违规/提示分档、算分封顶。
- **端到端**：护肤号(≤2 条且无误报封号级)、金融号(保本/加微信 命中)、旅游号(不触发任何行业包)。

---

## 14. 风险与待定

- **词边界启发式够不够** → 先启发式，反例测不过再评估 jieba。
- **法规时效**（三品一械等会变）→ 规则包带 `version` + 更新日期，集中维护。
- **赛道错分** → 多赛道 + 通用包兜底；长尾走 LLM，降低单点风险。
- **LLM 裁判成本** → 只处理候选子集 + 诊断才开；创作闸门默认不开 LLM。
