Cadence · UI 重构设计定稿

对象驱动、无侧栏导航 —— 前端重构的设计源文件。分阶段实现(UI·1–UI·12),此文件为对照基准。

核心理念

导航单位是「你的东西(对象)」,不是「功能」。对象只有三类:对标博主 · 我的账号 · 草稿。
移动方式:首页 Hub(选一个对象)→ 对象页(它的功能都在里面)→ 面包屑返回首页。没有常驻左侧功能栏。
原来侧栏里的「档案 / 对标分析 / 对标创作」本质都是「针对某个博主做的事」,收进博主页(段控「档案 | 对标分析」+「用它创作」按钮);我的账号同构(段控「档案 | 体检」)。

已定决策

· 公众号(每日早报)本轮独立保留、不动(异类工作流,不进对象驱动工作台)。
· 小红书 / 抖音合并:平台从「进门二选一的门」降成对象的属性,去掉选平台页;平台只靠对象卡片上的标签区分,本轮不做顶栏平台筛选(以后博主多了再按需加筛选/分区)。
· ⌘K 快速跳转:本轮只留占位,后面再做。
· 品牌 = Cadence(层叠菱形 logo);主色青绿 #0d7361,标题字体 Space Grotesk + PingFang SC。
