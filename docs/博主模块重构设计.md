# 博主模块重构设计

> 合并问题清单 #1 / #2 / #3 / #5。这四件事互相牵连(共用 user_info 解析、共用博主资产页、动到蒸馏选材),
> 作为一次"博主模块重构"统一设计,分两期落地。#4(评论数键)已单独修复上线。

## 背景(各问题)
- **#1**「创建博主」按钮不该在数据采集页;采集只**选**博主,创建挪到「博主资产」。
- **#2** 编辑博主只允许改 手动标签 / 备注 / 领域赛道;昵称/主页/头像/粉丝数等只读;要更新这些 → 新增「刷新博主」按钮重拉资料覆盖。
- **#3** 博主信息新增「笔记总数」;口径:**尽量从平台资料(user_info)解析**,拿不到则留空 / 只显示「已采集 N 条」。
- **#5** 砍掉「采集历史(批次)」视图,博主详情改为**笔记按类型(图文/口播/非口播)分类 + 点进看单篇详情**;连带**蒸馏选材从"选采集批次"改为"从笔记池/命名快照选材"**。

## 一、数据 / 后端
- `BloggerProfile` 加列 **`note_total: int|None`**(平台侧笔记/作品总数,解析不到则 NULL)。迁移 `0009_blogger_note_total.py`(幂等加列)。
- **user_info 解析**(给 #2 刷新 + #3 总数共用):新增 `extract_user_profile(payload, platform) -> {display_name, avatar_url, follower_count, note_total}`。
  - 抖音:`aweme_count`(作品数)较稳;复用 `normalize_douyin_user_info`。
  - 小红书:尝试常见键(`notes_count`/`note_count`/`tab_public.notes`…);**拿不到就 None**(老实留空)。
- **刷新博主资料** `refresh_blogger(db, tenant_id, blogger_id)`:建 client → `get_user_info(homepage_url, external_id)` → `extract_user_profile` → 覆盖 `display_name/avatar_url/follower_count/note_total`(空值不覆盖)。
  - API:`POST /bloggers/{id}/refresh-profile`(同步,轻量,一次 user_info 调用)。
  - 采集时(`run_blogger_collection` 开头已调 `get_user_info`)顺带解析 `note_total` 写入,免得单独再拉。
- **蒸馏选材改造(#5 连带)**:蒸馏入参从 `collection_run_id` → **`post_ids: list[int]`(从笔记池选)或 `snapshot_id`**。
  - 采集批次(`BloggerCollectionRun`/`BloggerCollectionPost`)**后端保留**(仍是采集留痕、增量/下架对账需要),只是**蒸馏不再依赖它选材**、前端不再展示批次。
  - 命名快照:把之前押后的「笔记池 + 命名快照」提上来(轻量:`BloggerSnapshot{name, blogger_id, post_ids}`,或先支持"手选 post_ids 直接蒸",快照后续加)。

## 二、前端
- **数据采集页**(#1):移除「创建博主」按钮;第 1 步只保留选择(列表 + 提示"没有就去博主资产创建")。
- **博主资产页**(#1/#3/#5)重构:
  - 头部:博主信息 + **笔记总数(note_total,无则显示"—")+ 已采集 N 条** + 「刷新博主」按钮 + 「创建博主」入口。
  - 主体:**笔记池按 content_subtype 分组**(图文 / 口播视频 / 非口播视频 / 未知)的笔记列表;点单条 → **单篇详情**(标题/正文/口播逐字稿/评论/各项数据,数据现成)。
  - 去掉「采集历史(批次)」与「蒸馏历史(按批次)」的批次中心展示;蒸馏入口改为"选笔记 → 蒸馏 / 存快照"。
- **编辑博主弹窗**(#2):昵称/主页/头像/粉丝数 改为**只读展示**;仅 手动标签 / 备注 / 领域赛道 可编辑;加「刷新博主」按钮(调 refresh-profile)。
- 类型:`BloggerProfile` 加 `note_total`;蒸馏请求改 `post_ids`/`snapshot_id`。
