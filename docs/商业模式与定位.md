# PubSync 商业模式与定位

> 定稿 2026-06-24。先定"卖给谁、解决什么、怎么收钱",技术实现据此推导。决策来自一次商业讨论。

## 1. ICP（理想客户）
靠内容获客的**个体专业户 / KOC**——保险经纪、教培老师、医美/健身/装修顾问、想做个人 IP 的人。
- 一个人管 1–3 个号（小红书/抖音为主）；
- 不会写爆款、没时间、不懂对标拆解；
- **内容直接关系收入**（涨粉 → 私域 → 成交/接单）；
- 价格敏感但愿为"能涨粉 / 省时间"付百元级月费。
- **范围：泛打所有个体专业户**（不先做垂直）。代价见 §7 冷启动。

## 2. 价值主张（一句话）
> "把你这行**头部博主的打法**，变成**你自己能日更**的内容——还告诉你差在哪、怎么补。"

## 3. 壁垒（为什么不是通用 AI 写作）
通用工具给"泛泛稿子"；我们的核心是：
- **对标蒸馏**：把某个真人头部博主的方法论提炼成可复用 Skill（贴其风格）；
- **账号诊断**：我的号 vs 对标，指出短板 + 可执行动作；
- **（进阶）Skill 训练**：用博主真实笔记把 Skill 练得更像，可量化提升（壁垒，非 MVP）。
个体户最缺的正是这个：知道"想做成谁",但不会拆解、不知道自己差在哪。

## 4. 交付形态：混合（平台自助 + 轻代运营）
- **主体**：自助 SaaS——选对标 → 采集 → 生成 → 改 → 发草稿；
- **轻代运营（高档增值）**：首次冷启动（帮做对标蒸馏 + 首批内容 + 诊断报告），或月度陪跑；
- 泛打无行业预置 → **用轻代运营的人力补冷启动**，跑通后沉淀成自助引导 SOP。

## 5. 变现：统一套餐 + 对标位按平台分桶
**计费结构（已定）**：一个统一订阅分 3 档；核心计费维度 = **对标博主数 × 平台**（一个对标位≈一次首采+持续增量刷新=最大成本，用户也好懂）。小红书位贵、给得少；抖音位便宜、给得宽。套餐越贵 = 更多小红书位 + 更多抖音 + 训练/诊断/代运营。
- **张力**：小红书是用户主战场又最贵 → 小红书必含、要卖出价；抖音便宜，作加量/增值。**别让想多用小红书的人被迫买最贵档**（中档就要给够小红书位）。

| 档 | 价(待校准) | 小红书对标位 | 抖音对标位 | 功能 | 采集COGS估 | 毛利 |
|---|---|---|---|---|---|---|
| 入门 | ~¥99/月 | 2–3 | — | 一次性蒸馏 | ~$5–7 | 紧(~50%) |
| 进阶 | ~¥299/月 | 8–10 | 解锁(宽松) | **+训练+诊断** | ~$12–18 | ~60% |
| 专业 | ~¥599+/月 | 20+ | 不限 | **+轻代运营** | 采集~$25+人力 | 靠人力定价 |
- **超量**：加购对标位——小红书 ¥3–5/个/月、抖音 ¥0.5/个。
- 维度别太多：对小白只暴露"能盯几个对标号 + 含不含训练/诊断"，平台差异藏在对标位里，不暴露 credit。
- 未选"按平台独立订阅"(SKU 多、决策摩擦)和"积分制"(对小白抽象)。

### 5.1 成本铁律（TikHub 真实价，已用 calculate_price 实测 2026-06-24）
| 平台 | 资料/列表/详情/评论 单价 | 折扣 | 一次采集 |
|---|---|---|---|
| **小红书** | **$0.01/次，全部，无折扣、无免费额度** | ❌ | 首采 $0.6–1.7 / 增量 $0.2–0.6 |
| **抖音** | **$0.001/次，多数可打折(量大到5折)** | ✅ | 首采 $0.1–0.2 / 增量 $0.02–0.05 |
- **小红书比抖音贵 10× 且零折扣** → 采集额度/对标位定价必须按小红书算，否则亏穿(进阶档若放开小红书采集，光 COGS 就能超过月费)。
- 生成≈免费(MiniMax M2.7 几分钱)、ASR 按视频(腾讯，另算)。
- **保毛利杠杆**：默认走"增量 + latest"(比首采便宜 5–8×)；首采/对标位封顶；入门档毛利紧→杠杆是小红书对标位数 + 增量频率。
- 落地差一层：**套餐 / 对标位额度(分平台) / 功能门控**（admin 已有 CostEvent 记账打底）。
- 省钱优化(待做)：`fetch_user_notes_page` 每页打多个 user_notes 端点，$0.01 下是真钱，第1页定最优端点后续只用它可省小红书列表费。

## 6. 对技术路线的重排
1. **业务级"涨粉 / 对标差距收敛"证据 > skill 级 SFS** —— 对 ICP 是核心卖点；差距收敛曲线(业务全流程回路③) 优先级上升。
2. **Skill 训练(SkillOpt) 降为进阶档卖点/壁垒,非 MVP** —— MVP 用一次性蒸馏的 skill 即可交付价值（PoC 已验证管线通、训练值得做但不挡 MVP）。
3. **冷启动体验是命门**："选个想学的博主 → 一键出首批内容 + 诊断报告"必须顺滑。
4. **合规扫描从附加升为刚需卖点** —— 保险/医美/教培是限流词/监管雷区，正好戳中。
5. **平台聚焦小红书/抖音**（ICP 主战场）；公众号早报偏"实体店"人群 → 先降级或另立。
6. **计费/额度层落地** —— 套餐、采集额度、分档功能门控。
7. **"对标任意博主"入口傻瓜化** —— 搜/粘贴链接全自动（已有搜索 + URL 定向采集）。

## 7. 风险与对策
| 风险 | 对策 |
|---|---|
| 个体户 churn 高、价格敏感 | 首周出第一批能发的内容 + 拿涨粉/差距收敛证据留存 |
| 泛打冷启动难(无行业预置库) | 轻代运营人力补 + "对标任意博主"通用入口 |
| 自助门槛(蒸馏/快照黑话) | UI 包装成"选个想学的博主→出内容"，藏黑话(已在简化) |
| 采集成本失控 | 采集额度卡死 + 超量计费 + 后台监控 |
| 轻代运营吃时间 | 只给高档 + 标准化冷启动 SOP |

## 8. 不做的事（本阶段）
- 不先做垂直行业预置库（选了泛打）；
- 公众号早报不作为 ICP 主线；
- 纯按量计费（与成本结构错配）；
- 重代运营/按效果分成（重服务、难规模）。

## 9. 下一步
据此推导 **MVP 范围**：冷启动顺滑(选对标→首批内容+诊断) + 一次性蒸馏创作 + 合规 + 计费/采集额度层；Skill 训练与差距收敛曲线排进进阶迭代。
