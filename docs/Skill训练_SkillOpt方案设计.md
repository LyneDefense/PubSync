# Skill 训练方案设计（基于 SkillOpt 思路）

> 起草 2026-06-20。目标：把"一次性蒸馏出的静态 Skill"升级为"可训练、且能给出可量化提升数据"的资产。
> 本文是方向性设计，未写代码。决策门控见每阶段「Gate」。

## 0. 一句话目标
对每个对标博主，产出一份**训练后的 `best_skill.md`**，并用**该博主自己从未参与训练的真实笔记（test 集）**量出"训练前 vs 训练后"的风格保真度提升数（例：62 → 78，+16 分），作为是否值得推广的依据。

---

## 1. 背景与现状
- **现状链路**：采集博主笔记 → 一次性蒸馏出 `skill_markdown`（`blogger_distillation/service/distillation.py`，静态，提炼完即定）→ 创作时把 skill 喂给 `xhs_creation/agent/*`，每篇靠 synthesis harness 自检修订（推理时成本）。
- **痛点**：skill 是"一次猜中"，没有反馈回路；好不好全凭蒸馏 prompt，无法度量、无法改进。
- **SkillOpt 是什么**：微软开源库（`pip install skillopt`，MIT）。不训练模型权重，而用"文本梯度"反复改写一份 `skill.md`；循环为 `rollout → reflect → aggregate → select → update → evaluate`，**validation-gated**（只有验证集变好才保留改写），产出精炼 `best_skill.md`。支持 Claude 后端。
  - 来源：https://github.com/microsoft/SkillOpt

## 2. 关键洞察：博主真实笔记 = gold 标签
SkillOpt 自带 benchmark 有客观答案；我们"模仿博主写小红书"没有标准答案。**但博主自己的真实笔记，就是"这个博主会怎么写"的黄金答案。** 于是把任务转成一个有监督设置：

- 给定博主真实笔记的**选题/标题**为输入，让 skill 生成一篇，去**对比该博主真实那篇**的相似度（风格、结构、语言 DNA）。
- 相似度越高 = skill 越"懂"这个博主。

这把"无标签"难题解掉了，是整套方案能成立的根基。

## 3. 数据划分（每个博主一套，**时间留出**）
要求该博主在架（status != delisted）笔记 **≥ 24 篇**（不足不训练，提示多采）。

**样本单位**：一篇笔记。输入 = 该笔记的「选题/标题 + 核心信息点」；**正文是 gold，只用于打分，绝不喂给模型**（否则是照抄）。

**为什么用时间留出而非随机分**：博主风格随时间漂移，我们要的是"像 ta **现在**怎么写"。随机分会把"未来笔记"混进训练 = 偷看 + 不对路。故按时间切：

| 集合 | 占比 | 时间位置 | 用途 |
|---|---|---|---|
| **train** | ~60% | 较早 | 蒸种子 skill + rollout few-shot 范例 |
| **val** | ~20% | 中间 | 验证门控（候选 skill 只有在 val 提升才保留） |
| **test** | ~20% | **最近** | 全程不参与训练，**提升数只在此集报告** |

**三道防泄漏（必须做）**：
1. **系列不跨集**：先按选题聚类，系列/同主题整簇分到同一集合（防 day1→train、day2→test）。
2. **评委范例只取 train**：SFS 评委的 few-shot 参考笔记只能来自 train，不能用 val/test。
3. **只给题不给正文**：rollout/eval 的"题目"= 选题 + 核心信息点，不泄漏正文。
- 还要按模态分层（图文/视频各集都有，因为一个 Skill 覆盖混合）。
- 小样本坑：24 篇 → test 仅 ~5，估计噪声大；PoC 时再跑几个随机种子分割报区间做交叉校验，但**时间留出为主口径**。

## 3'. 博主更新笔记后如何处理（划分不冻结 + 近期窗口 + 批量重训）
新笔记代表博主**当前**方向，最值钱。原则：
- **每次重训按当前笔记池重新切**（时间留出），划分不固化：新笔记成为新的 test，旧 test 滚入 train/val。
- **近期窗口**：只用最近 N 篇 / X 个月训练（config 可调，如最近 50 篇 / 6 个月），丢弃过旧风格，避免把 Skill 往回拽。
- **重训触发（批量，别每篇都训）**：① 笔记池新增 ≥X 篇 或 ≥Y%（标杆演化，回路②）；② **底座/生成模型变更**（见 3''）。任一触发 → 先复验、必要时重训。
- **种子**：重训时用当前 train 重新蒸一个干净种子；可同时从上一版 best_skill 热启动，取在新 val 上更好者。
- **横比警告**：每次重训 test 集都变（新近期笔记 ≠ 同一张卷子），**两次重训的 SFS 提升数不能直接横比**；单次内"种子 vs best"仍有效，**跨时间进步看业务级指标（对标差距收敛），不是裸 SFS**。

## 3''. skill 与底座绑定（换大模型 → 偏差）
训练用"某冻结模型"的行为当梯度，故 skill 是**绑定该底座**的资产。skill 含两类成分：
- **A 风格事实（与模型无关）**：标题公式、口头禅、结构——可迁移。
- **B 模型纠偏（与模型强相关）**：补该模型怪癖的指令——换模型可能失效甚至有害。
处理：
- **钉死底座+版本**：`SkillTrainingRun` 记录训练所用生成模型与版本；skill 不是放之四海皆准。
- **换底座=触发"先复验、必要时重训"**：用 新模型+旧 skill 跑 test 多指标判定，没掉留用、掉了重训（复用 4' 判定）。同家小升级漂移小；跨家大换血多半要重训。
- **训练偏向可迁移**：反思阶段倾向多写"博主怎么写"(A)、少写"怎么哄这个模型"(B)，让 skill 更扛换模型。
- **创作配套**：默认用 skill 训练时的底座，要换走复验。

## 4. 度量体系（成败核心，必须先做扎实）

**原则：全程零人工标注。** 面向客户的产品不能要求任何人（客户或我们）手动给样本打分来背书评委。可信度改由"博主真实笔记自身"作锚点自动建立。度量分三层，越靠前越客观、越难被刷。

### 4.1 客观风格指纹距离 StyleDist（确定性，无 LLM、无人工）
从博主真实笔记建一份**风格指纹**（纯文本统计，可复现）：标题模式正则命中分布、句长/段落数分布、emoji 率、话题标签数、开篇类型分布、高频 n-gram/口头禅词表、书面 vs 口播比例等。对候选算它到该指纹的距离（越近越像）。
- 完全确定性，无模型调用、无人工；可作奖励分量，也是评委的"客观对照"。
- 复用已有：`blogger_distillation/analysis.py`、`quality.py`、`compliance`。

### 4.2 风格归属判别 StyleAttribution（自动，有客观标签）
把候选与"该博主真实笔记 / 其它博主真实笔记"一起做归属：嵌入向量最近邻 / 轻量分类器判"这像不像该博主"。**因为我们知道每篇的真实作者，标签是客观的，不需要人标。** 报 AUC/准确率。

### 4.3 可辨别度 Detectability（自动，有客观标签，最难刷）
判别器看"候选 + 一篇该博主真实笔记"，指认哪个是 AI 写的。**我们知道谁真谁假，标签客观。** 判别越难（越接近随机 50%）= 越像真人。比 rubric 更难被 reward hacking。

### 4.4 LLM 评委 SFS（0–100，rubric 细分，用于反思的"文本梯度"）
一个冻结的强模型评委，按 rubric 逐维打分 + 给理由（理由喂给反思阶段当文本梯度）：选题契合(20)/标题钩子(20)/正文结构(25)/语言DNA(25)/合规(10，硬门)。
- 训练阶段需要它的"理由"来指导改写，所以保留；但它**不单独决定"提升数"**——见 4.5。

### 4.5 评委凭什么可信：自动锚点电池（替代原 Gate 0 人工打分）
不让人打分，改让评委通过一组**用现有数据就能自动构造的 sanity 测试**（锚点电池），全自动、可重复、客户无感：
1. **单调分离**：喂"该博主真实笔记 → 应高分"、"其它博主真实笔记 → 应低分"、"通用 AI 套话 → 更低"、"打乱/胡言 → 最低"。评委分数若不能单调拉开，就是不可信 → 调 prompt/模型，不进训练。
2. **判别校准**：在已知真假的对子上测评委判别准确率（4.3）。在"真 vs 真"对子上应接近 50%（不该乱猜），在"真 vs 烂"对子上应接近 100%。
3. **与客观层一致性**：评委 SFS 必须与 StyleDist/StyleAttribution **方向一致**（同升同降）才采信；不一致时以客观层为准。

> 结论：最终对客户展示的"提升数"由**客观层(4.1–4.3)主导 + 评委(4.4)佐证**，三者方向一致才报。任何人都不用手动打分。

## 4'. 量化口径与判定规则（怎么算成数字、怎么判好坏）

### 4'.1 每个指标怎么变成一个数（batch 级，夹在 floor/ceiling 之间）
先用 train 笔记建风格指纹/质心；指标都在**一批生成结果**上算，并对两个锚点定标：
- **ceiling 天花板** = 博主 test 真笔记（真人也非满分，有自然波动）。
- **floor 地板** = 见 4'.1a。

#### 4'.1a floor 的来源（两种，都现成）
1. **其它博主真实笔记**（系统已采，多个 BloggerProfile）：当"像小红书但不像本博主"的地板。**必须挑同/近赛道**（`niche`），否则归属判别太易、虚高"差距缩小"。只有 1 个博主时退到第 2 种，**不跨租户取数（隐私）**。
2. **无 skill 白板 AI**：同选题、裸 prompt、不给任何 skill 生成一篇 = "AI 啥都不懂时写成啥样"。训练后若不比白板更像博主就是白训。永远可得。

> 真实尺子链：`白板AI(floor) < 种子skill(基线) < 训练后(?) < 博主真笔记(ceiling)`。头条数是"训练后 vs 种子"，floor/ceiling 用来定标、防赛道难度忽悠。

| 指标 | 计算 | 取值 |
|---|---|---|
| StyleDist | 特征向量(标题公式正则命中/句长段落分布/emoji率/n-gram词表/开篇类型/书面口播比) → 生成批聚合向量与指纹的 `1 - 距离` | 0–100（越高越像）|
| StyleAttribution | embedding 最近质心分类，判成"本博主"的比例 / 软margin | 准确率 0–1 |
| Detectability | 判别器看「候选+真笔记」乱序猜 AI，多模型多次取均 | 抓中率 50–100%（越低越好）|
| SFS | 评委 5 维 0–100 + 理由（理由=文本梯度，分数仅佐证）| 0–100 |

### 4'.2 合成判定（非简单平均）
1. 同一 **test** 集比 种子 vs 训练后，各指标都算。
2. **归一为"缩小到天花板的差距比例"**：`gap_closed = (训练后 − 种子)/(天花板 − 种子)`。
3. **噪声护栏**：生成随机 → 先把种子跑两遍测"噪声带"，训练后须超过它；再对 test 题目 bootstrap，**提升 CI 须排除 0**。
4. **多指标一致（信任规则）**：仅当 **StyleDist↑ 且 Attribution↑ 且 Detectability↓** 三客观层同向且超噪声、SFS 不反向，才判"训好了"；**分歧以客观层为准**。
5. **防刷内建**：长度惩罚 + 合规硬门(命中限流词该 rollout 判负) + Detectability 兜底(SFS 涨但判别器照样识破=化妆式作弊，打折)。

### 4'.3 训练过程诊断（不止看终值）
- **val 曲线**：升后平台=健康；平=学不动；val 升但 test 平=过拟合；震荡=不稳(调小文本学习率)。
- **拒绝率**：候选被 val 门控拒太多 → 反思没找到真改进。

### 4'.4 最终判定（部署 or 丢弃）
- **训好了** → 部署训练后 skill；展示"在从未见过的最近 N 篇上缩小到真人风格 X% 差距(StyleDist +a/归属 +b/难辨度 +c)，多指标一致"。
- **没改进/可疑** → 保留种子 skill，老实报"未拿到可靠提升"。**绝不推未验证结果。**

### 4'.5 防 Goodhart：训练奖励 ≠ 判定指标
训练时优化 **SFS + StyleDist**（密集、好驱动改写）；最终判定靠 **Detectability + Attribution**（留出、客观、难刷）。故意错开，使模型无法"专刷要上报的那个数"。

## 5. 训练循环（SkillOpt 适配版，每个博主）
1. **种子 skill** = 现有一次性蒸馏输出（train 集蒸）。
2. for epoch in 1..E：
   - **Rollout**：抽 K 个 train/val 选题，用当前 skill（模型冻结）各生成一篇。
   - **Reflect（文本梯度）**：评委给每篇打 SFS + 指出差距；反思 LLM 读"低分案例 + 差距 + 对应真实笔记"，产出对 skill 的**具体改写建议**（如"标题缺『数字+悬念』公式""正文应痛点开篇，你写成了功能罗列"）。
   - **Aggregate + Update**：合并建议 → 候选 skill。
   - **Validation gate**：候选在 **val** 上测 SFS；优于当前最优（超噪声阈值）才采纳，否则丢进 rejected-edit buffer（避免重复同类无效改写）。
3. 取 val SFS 最高者为 `best_skill.md`。
4. **最终评估**：在 **test** 集上跑 种子 vs best 的 SFS + Detectability + 给几对并排样例 → 这就是"可量化提升数据"。

> 早停：val SFS 连续两轮无提升即停，省成本。

## 6. 适配性评估与部署形态（库 vs 自研 / 独立 vs 嵌入）

### 6.1 SkillOpt 实测形态（microsoft/SkillOpt）
- 可 `pip install` 当库用 + CLI + 可选 Gradio WebUI；纯 Python、纯 API 调用（不拖 torch/vllm）；**Python 3.10+**（我们 3.11 兼容）；内置 `claude_chat` 后端；MIT。
- 自定义任务需按目录约定写 `skillopt/envs/<name>/{dataloader.py, rollout.py, initial.md}`，由**它**驱动循环、回调你的 rollout（控制反转）。
- 成熟度：v0.1.0（2026-06-02 首发，很新）；8.4k★ 但 18 open issues。

### 6.2 适配结论：算法适合，库不适合当产品运行时依赖
- ✅ **思路完全契合**：text-space skill 优化、Claude 后端、validation-gated，正是我们要的。
- ⚠️ **耦合冲突**：我们的**生成器(Claude 客户端)+奖励器(StyleDist/分析/评委)与 DB/模型/笔记深度耦合**；SkillOpt 要当主控并按 env 约定回调，意味着要么把 FastAPI/SQLAlchemy 拖进它进程、要么重写打分逻辑。
- ⚠️ **v0.1.0 太新** + 自带 WebUI/benchmark 包袱，不宜常驻后端；且其循环 ≈ 我们 synthesis harness 已有的 80%。

### 6.3 部署决策（分阶段）
| 阶段 | 形态 | 做法 | 理由 |
|---|---|---|---|
| **PoC(M1)** | **独立/离线** | 本机或一次性容器跑 SkillOpt，写最小 `envs/xhs_blogger/`：rollout 调 Claude + 自包含打分器(读笔记 JSON dump，不连线上 DB) | 零污染生产，用现成循环最快验证提升是否成立；可用其 WebUI 看曲线 |
| **生产(M2+)** | **嵌入式·自研精简循环** | 把 rollout→reflect→val门控→select 移植进 harness，做 `run_skill_training_task`(RQ)，藏在 `SkillTrainer` 接口后；**不把 skillopt 当运行时依赖** | 奖励器/生成器必须与 DB/Claude/成本/进度/取消同进程；规避 v0.1.0 风险；复用已有机件 |

- **兜底**：若自研循环复现困难，退为"独立离线训练容器"（批处理读 dump → 写 best_skill.md，打分器在容器内重写一份）；代价是逻辑重复+多服务，仅在必要时采用。
- **隔离**：无论哪种，训练器都藏在 `SkillTrainer.train(seed, datasets) -> TrainResult` 接口后，SkillOpt 库只是可替换实现之一。

### 6.4 接入方式 & 评委归属（SkillOpt 不带通用评委）
**SkillOpt 只提供"优化循环 + 反思改写"，打分由你的 `rollout.py` 返回**——它 6 个内置 benchmark 各自带打分器，自定义任务的奖励=我们给的。分工：**SkillOpt 管循环，我们管生成器+打分器。**
- 接法：写 `skillopt/envs/xhs_blogger/`：
  - `dataloader.py`：吐题目=（选题+信息点），读笔记 JSON dump（PoC 不连线上 DB）。
  - `rollout.py`：①当前 skill 调 Claude 生成；②我们的打分器对真笔记打分 → 返回 **标量奖励 + 文字点评**（点评=反思阶段的文本梯度，光给数改不好）。
  - `initial.md`：我们蒸的种子 skill；backend：内置 `claude_chat`。
- **两条边界**：① train/val 的优化与门控交给 SkillOpt（val 当其验证集）；② **最终 test 判定我们在 SkillOpt 之外做**（第 4' 节多指标+噪声+CI 规则），不把成败交给它内部那个标量——防 Goodhart，评委始终在我们手里。

## 7. 集成架构（PoC 通过后）
- **离线训练任务**：`run_skill_training_task`（RQ task，与采集/蒸馏同款）。发 `record_task_event` → 在 LiveProgress 面板显示 epoch、val SFS 曲线、当前最优。
- **数据模型**：新增 `SkillTrainingRun`：`blogger_id, seed_skill_id, best_skill_md, baseline_sfs, best_sfs, lift, detectability_before/after, val_curve_json, test_report_json, epochs, cost_tokens, status`。Alembic 迁移。
- **存储**：best_skill 作为一个新的 skill 版本挂在 run 上；创作时可选"用训练后的 skill"。
- **模型/参数**（入 `config.py` + admin 可调）：评委模型、生成模型、反思模型；E（epochs）、K（rollouts）、min_notes、val/test 比例、噪声阈值、预算上限。
- **成本记账**：训练每次 LLM 调用挂进 CostEvent，run 汇总 `cost_tokens`，后台可见。
- **前端**：蒸馏页某个 skill 上加「训练这个 Skill」→ 训练进度 → 完成后一张 **「训练前/后对比卡」**：`test SFS 62→78（+16）`、`可辨别度 81%→58%`、几对并排示例。这张卡就是你要的"可量化效果提升数据"。

## 8. 成本模型（基于当前 Claude 价目，2026-05-26）
单价/百万 token：Opus 4.8 $5/$25、Sonnet 4.6 $3/$15、Haiku 4.5 $1/$5。缓存命中读 ~0.1×、写 1.25×(5m)/2×(1h)。

**用量**：E=4、K=6、val=4、test=5 → **~95–130 次调用/博主**（生成~50、SFS~60、判别~10、反思4、改写4、蒸种子2）。小红书笔记短，单次约 3k 输入/0.6k 输出 → **≈ 40万输入 + 8万输出 token/次**（未开缓存）。

**成本/次（每博主）**：
| 搭配 | 估算 |
|---|---|
| 全 Sonnet | ~$2.4 |
| Sonnet 生成 + Opus 评委（推荐：评委冻结要准） | ~$3.2 |
| 全 Opus | ~$4.0 |

- **缓存**(skill/few-shot/rubric 反复用，读 0.1×)可再砍 30–50% 输入 → 实际多落 **$1.5–3**；
- **adaptive thinking** 开启会涨输出 1.5–2× → 上界 **~$5–6**。
- **结论：一次训练 ≈ $2–6/博主，按 $3–4 估稳**。一次性离线成本，之后创作复用、可摊薄。100 博主首训 ≈ $300–400。
- 每次调用挂 `CostEvent`、`SkillTrainingRun` 汇总 `cost_tokens`，**PoC 跑完即有准数**；全程受预算护栏，超预算早停留最优。

## 9. 风险与对策
| 风险 | 对策 |
|---|---|
| 评委不可信/可被刷 | **零人工**：自动锚点电池(4.5) + 客观层(StyleDist/归属/判别)主导，评委仅佐证、全程冻结 |
| 过拟合 val | test 集严格隔离，**只报 test 提升** |
| Reward hacking（skill 变啰嗦/堆词） | 长度惩罚 + 合规硬门 + Detectability 兜底 |
| 成本失控 | E/K 上限、val 早停、CostEvent 预算护栏 |
| 小样本博主 | 强制 ≥24 篇，否则不训练 |
| SkillOpt 库尚新（v0.1.0） | 仅作 PoC 依赖，藏接口后，可换自研循环 |
| 选题泄漏 | 划分按选题去重 + 时间分层；rollout 只给选题不给正文 |
| **换底座致 skill 偏置** | skill 绑定底座(钉版本)；换模型→先复验后必要重训；训练偏向可迁移的风格事实(见 3'') |

## 10. 分阶段计划与决策门控
- **Phase 0 · PoC（离线脚本，不进产品，零人工标注）**
  1. 选 1–2 个 ≥24 篇的干净博主。
  2. 建客观度量层（StyleDist/归属/判别）+ LLM 评委；跑**自动锚点电池(4.5)**自检评委——真实笔记>其它博主>套话>胡言能否单调拉开、判别校准是否合理。全自动、无人参与。
  3. baseline：种子 skill 在 test 选题上生成 → 客观层 + 评委 + Detectability。
  4. 跑 SkillOpt 训练 → best skill → test 上再测。
  5. 产出对比报告 + 并排样例（客观层主导、评委佐证、方向一致才报）。
  - **Gate 1**：test 上提升真实、且客观层与评委方向一致 → 继续；否则停（便宜地学到"不值得"）。
- **Phase 1 · 后端**：`SkillTrainingRun` 模型 + 迁移；训练器 RQ 任务 + 事件流 + 成本记账；config/admin 参数。
- **Phase 2 · 前端**：「训练 Skill」入口 + 进度 + 训练前/后对比卡；创作可选训练后 skill。
- **Phase 3 · 迭代**：打磨评委/奖励；视情况把循环移植进自家 harness。
