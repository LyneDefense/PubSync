# 博主档案页 · 建档与更新流程设计

> 上位文档：[[博主画像模块_全景设计]]（五层立体画像总纲）、[[蒸馏产出重定义_方案]]（④⑤层产出定义）。
> 本文回答产品级问题：**有哪些流程、每步要什么数据、页面怎么展示、数据怎么更新、缺数据怎么提示**。

## 一、核心模型：物理信息 → 分析信息

用户的定义（本文遵此）：**第一步把博主的信息构建全（物理层），再用这些信息构建画像（分析层）。档案主页承载两层。**

| 层 | 内容 | 来源 | 成本 | 更新方式 |
|---|---|---|---|---|
| **物理信息**（平台事实） | 身份资料；笔记池（每篇的标题/类型/发布时间/赞藏评浏览；详情级另有正文/转写/图内文字/评论） | TikHub 拉取 | 花钱 | 增量拉取，落库持久化 |
| **分析信息 · 确定性** | 数据统计、轨迹曲线、阶段划分 | 从笔记池计算 | 免费 | 物理层一变**自动重算** |
| **分析信息 · LLM** | 爆文归因、认知、方法（创作画像） | LLM 蒸馏/分析 | 花钱 | 不自动重跑，**标过时提示** |

关键规则：
- 分析永远从库内物理层算，**不直接调 API**——物理层是唯一事实源，换算法重算零成本。
- 与 [[蒸馏产出重定义_方案]] 术语对照：物理层+确定性分析 ≈ 「博主档案（账号事实，唯一）」；LLM 蒸馏产物 ≈ 「创作画像（可多个）」。

## 二、笔记池：一个池、两个详情级别

复用 `BloggerPost`，加 `detail_level` 字段，不新建表：

| 级别 | 覆盖 | 字段 | 来源 | 单价 |
|---|---|---|---|---|
| **list（列表级）** | 全部笔记 | external_id/note_key/标题/类型/**发布时间(create_time)**/赞/藏/评/分享/**浏览**/xsec_token | 列表翻页 | ~$0.01/页(20篇) |
| **full（详情级）** | 被选中蒸馏的笔记 | 上述 + 正文/转写(ASR)/图内文字(视觉)/评论 | 详情管线(note_pipeline) | ~$0.02+GLM/篇 |

- 列表级实测字段齐全（阿璐 301 篇：`create_time`/`likes`/`collected_count`/`comments_count`/`view_count` 全非零，`reached_end=True` 翻到第一篇，~$0.16）。
- list → full 只升不降；full 笔记重复出现在列表时只刷新互动数。
- 现状改造点：现在列表候选**没选中就丢弃**，要改成全部 upsert 为 list 级；`parse_user_notes` 需补解析 `create_time` → `published_at`。

## 三、建档流程（一站式长任务）

入口（已拍板）：添加博主后进入**空态档案页**（五区皆「未构建」骨架），中央主按钮「**构建博主画像** · 预计 ¥X · 约 Y 分钟」——建档是显式动作，不随添加自动扣费。点击后触发长任务，阶段串行，**每完成一个阶段，档案页对应区块立即点亮**（渐进呈现，复用 record_task_event / LiveProgress）。

| 阶段 | 做什么 | 调用/数据 | 产出（点亮区块） | 成本(约) | 耗时 |
|---|---|---|---|---|---|
| 1 建身份 | `get_user_info` 拉主页 | 1 次 | 身份卡（昵称/粉丝/note_total/简介/赛道） | ¥0.07 | 秒级 |
| 2 建笔记池 | `get_user_notes` 全量翻页（cap 600 / 80 页硬顶） | ~篇数÷20 次 | 笔记池（全部 list 级）+ `reached_end` 标记 | ¥1.2 (300篇) | <1 分钟 |
| 3 算统计+轨迹 | 纯计算：均值/比率/模态分布/**真·全量发布节奏**；时序曲线+基线+爆发检测+阶段划分 | 0 次 API | 数据面板 + 成长轨迹 | 免费 | 秒级 |
| 4 升级详情 | **最新 80 篇**（不足 80 全采；数量 admin 可调）→ 走现有 note_pipeline（详情/评论/视觉/ASR），已 full 的跳过 | ~160 次 + GLM | 笔记池 80 篇变详情级 | ¥11.5 + GLM 数元 | 6–8 分钟(并发4) |
| 5 自动蒸馏 | 用这 80 篇自动建快照「默认画像·最新80」→ 蒸馏 | LLM | 创作画像（认知+方法） | ¥1–3 | 2–5 分钟 |

- 全程约 **12–18 分钟、¥15–20**；其中阶段 1–3 只要 **~¥1.3、1 分钟内**——页面很快就有"身份+数据+轨迹"可看，贵的 4–5 在后台继续。
- 长任务最佳实践（一键到底 ≠ 干等到底）：**后台运行可离开**（RQ 任务，刷新/切页可恢复——现有能力）、**渐进点亮**、**分段进度**（"详情采集 37/80"粒度事件，LiveProgress 已有）、**完成通知**（页内 toast/徽章，小补）、**取消保留已完成阶段**（cancel_event 已有）、**断点续跑**（重试只补没做完的，note_key 命中的详情不重花钱，已有）。
- 爆文**归因**（LLM）不在建档默认链里：轨迹区提供「运行归因」按钮按需触发（~¥0.5），避免每个博主都花这笔。
- 失败处理：阶段 1 失败 → 整体中止（多半链接错/风控），可重试；阶段 2 中断 → 保留已翻部分、`reached_end=False`；阶段 4/5 失败 → 前 3 阶段成果保留，档案页对应区块显示失败态可单独重试。
- 成本透明：任务开始事件里给预估（复用 collect-estimate 机制），任务可取消。

## 四、档案主页（展示）

一页五区，每区自带状态徽章（见线框图）：

1. **身份卡**（头部）：头像/昵称/平台/赛道/粉丝/note_total + 「资料 X 天前更新」+「更新档案」按钮。
2. **数据面板**：均赞/藏赞比/互动率(赞藏评÷浏览)/视频vs图文均赞 等指标卡 + 「笔记池更新后自动重算」说明。
3. **成长轨迹**：内容表现曲线（时序）+ 阶段 chips（冷启动/爬坡/爆发点/平稳）+ 覆盖徽章（已翻到第一篇 / 仅最近 N 篇）+ 归因卡（未运行则给 CTA）。
4. **创作画像**：画像卡（名称·基于哪批笔记·蒸馏时间·覆盖车道）+ 过时徽章 + 「重新蒸馏」「蒸新画像」；点进去看完整认知/方法（现有报告/skill 页演进）。
5. **笔记池**：列表（每篇：标题/时间/互动/`详情级|列表级`徽章）+ 计数汇总 + 「增量更新」按钮。

状态徽章统一五态：`未构建`(灰) / `构建中`(蓝,带进度) / `已构建·时间`(绿) / `可能过时`(黄) / `失败·重试`(红)。

## 五、数据更新策略

| 数据 | 触发 | 做法 | 成本 |
|---|---|---|---|
| 身份 | 手动「更新档案」；增量更新时顺带 | 1 次 get_user_info | ¥0.07 |
| 笔记池·增量 | 手动（默认更新方式） | 从最新往回翻，**连续一整页都是已有笔记即停**（近期互动顺带刷新——正在活跃变化的恰是近期笔记） | ~¥0.2–0.4 |
| 笔记池·全量校准 | 手动；或距上次全量 >30 天时提示 | 重新翻到底，校准老笔记互动、捕捉老笔记翻红 | ~¥1.2 |
| 统计+轨迹 | **自动**：笔记池任何变更后重算 | 纯计算 | 免费 |
| 归因 | 手动按钮；笔记池大更新后标过时 | LLM 一次 | ~¥0.5 |
| 创作画像 | **不自动**。过时条件：新增笔记 ≥20 篇 或 距上次蒸馏 >60 天 → 黄徽章提示「重新蒸馏」 | 蒸馏 | ¥1–3 |
| 详情级升级 | 蒸馏/换快照时按需补采（note_key 命中已 full 的复用） | note_pipeline | 按篇 |

新鲜度阈值（身份 7 天 / 笔记池 14 天 / 画像 20 篇 or 60 天）进 config + admin registry 可调。

## 九、已拍板决策（2026-07-03）

1. **默认详情级样本 = 最新 80 篇**（不足 80 全采；用户 2026-07-03 修正：默认画像代表博主"现在"的创作状态，取最新而非高赞；"她凭什么火"由轨迹层回答）。80 比 50 的核心收益是**少数车道覆盖**（口播等车道更易凑够 `distill_min_samples_per_subtype`）。数量进 config（`dossier_default_full_count`）admin 可调。轨迹层就绪后可加「爆发期画像」选材。
2. **一键到底，阶段间不打断确认**：靠「按钮上标预估费用/时长 → 后台运行可离开 → 渐进点亮 → 分段进度 → 完成通知 → 取消保留已完成 → 断点续跑」承接长任务体验（见第三节）。
3. **建档触发 = 档案页空态主按钮「构建博主画像」**：添加博主不自动扣费，显式一键构建。
4. **前端不出现任何金钱/费用文案**（用户 2026-07-03 拍板）：按钮只写动作与耗时预期，不标价；费用监控是后续独立功能（后台 CostEvent 记账已在跑，仅管理端可见）。本文此前的 ¥ 标注仅作内部成本参考。

## 十、手动操作入口与级联规则

「一键到底」是主路径；每个区块右上保留细粒度入口，面向重拉/重蒸/补采场景。**级联总原则：免费的（确定性计算）自动重算；花钱的（LLM）只标过时、绝不自动重跑。**

| 手动操作 | 入口 | 直接效果 | 自动级联（免费） | 对花钱产物的影响 |
|---|---|---|---|---|
| 更新档案（身份） | 身份卡 | 资料/粉丝数刷新 | — | — |
| 增量更新笔记池 | 笔记池区 | 新笔记入库+近期互动刷新 | 统计/轨迹重算 | 新增≥阈值 → 画像/归因标过时 |
| 全量校准笔记池 | 笔记池区（低频） | 老笔记互动校准、翻红捕捉 | 统计/轨迹重算 | 同上 |
| URL 定向补采 | 笔记池区 | 单篇入池（详情级） | 统计/轨迹重算 | —（单篇不动摇画像） |
| 删除/排除笔记 | 笔记列表 | status=excluded | 统计/轨迹重算 | 在画像样本内 → 画像标过时 |
| 重新蒸馏（默认画像） | 画像区 | 同快照重蒸，版本**覆盖** | — | 清除画像过时标 |
| 蒸新画像（换选材） | 画像区 | 新画像**并存**（多画像） | — | — |
| 运行/重跑归因 | 轨迹区 | 归因假设更新 | — | 清除归因过时标 |
| 重建档案（整链重跑） | 档案页菜单（罕用） | 阶段 1–5 重走（已 full 的详情复用不重花） | 全部重算 | 全部重建 |

配套规则：
- **任务互斥**：同一博主同时只允许一个构建类任务（建档/增量/全量/蒸馏）在跑，其余入口置灰并提示「有任务进行中」——防并发写池、防重复扣费。
- **画像记"基于版本"**：画像记录蒸馏时刻的样本（快照 post_ids，已有）+ 当时笔记池状态（时间戳/计数）；过时判定 = 池最后变更晚于蒸馏时间 且 新增/剔除超阈值。
- **过时 ≠ 失效**：标过时的画像照常可用（创作/选题不中断），徽章只是提示"有更新可纳入"。
