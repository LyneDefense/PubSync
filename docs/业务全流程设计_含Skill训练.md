# PubSync 社媒业务全流程设计（含 Skill 训练）

> 起草 2026-06-20。总纲：把现有各模块串成一个**可度量、能自我改进的闭环**，并标清 Skill 训练在其中的位置。
> Skill 训练内部细节见 [Skill训练_SkillOpt方案设计.md](Skill训练_SkillOpt方案设计.md)，本文只讲它如何与全流程衔接。

## 0. 客户到底要什么（北极星）
客户开了自己的小红书/抖音账号，想**长得更快**。PubSync 帮他做三件事，并且每件都能拿出数：
1. **学标杆**：把对标博主的成功方法论提炼成可复用的 Skill。
2. **照着写**：用 Skill + 自己账号的短板，自动产出"像标杆、又补自己短板"的内容。
3. **看进步**：用"我的账号 vs 标杆的差距收敛"和"发布内容的真实互动趋势"证明在变好。

**北极星指标 = 我的账号对标差距收敛 + 发布内容互动增长。** Skill 训练是把这两个指标往上抬的杠杆，不是孤立功能。

## 1. 端到端闭环全景（文字流程图）

```
            ┌──────────────────────── 标杆侧（学谁）────────────────────────┐
 找对标博主 → 数据采集 ──→ 博主笔记池 ──→ 蒸馏(Skill 种子) ──→ ★Skill 训练(best_skill)
  (博主资产)   (增量/URL)    (按类型)        (选笔记→一个Skill)    (本文重点·离线训练)
                                                                      │
            ┌──────────────────────── 自己侧（我是谁）──────────────────┐ │
 登记我的账号 → 采集我的内容 ──→ 对标诊断 / 诊断我的 ──→ 差距报告+动作清单   │ │
   (我的账号)     (缓存,手动刷新)     (逐维度对比)         (短板/优势/增长动作)│ │
                                                              │           │ │
                                                              ▼           ▼ ▼
                                              创作(对标博主创作 / 自由创作)
                                              输入 = best_skill + 我的差距 + 选题
                                                      │
                                              合规检查 + 对标对比(创作内 critic/benchmark)
                                                      │
                                              发布草稿(审阅/复制/发布)
                                                      │
                              ┌──────────── 反馈闭环（三条回路）────────────┐
                              │ ① 真实互动回流 → 在线奖励 → 再精炼 Skill      │
                              │ ② 增量采集标杆 → 标杆演化 → 重蒸/重训          │
                              │ ③ 周期性再诊断 → 差距收敛曲线(业务级可量化)   │
                              └──────────────────────────────────────────────┘
```

## 2. 各阶段：输入 → 产出 → 谁消费（资产流）
| 阶段 | 模块(现状) | 输入 | 产出（资产） | 下游消费方 |
|---|---|---|---|---|
| 选标杆 | 博主资产 ✅ | 主页链接/搜索 | BloggerProfile | 采集、蒸馏 |
| 采集 | 数据采集 ✅ | 博主 + 配置 | BloggerPost 笔记池(含转写) | 蒸馏、训练、诊断对比 |
| 蒸馏 | 蒸馏 ✅ | 选中笔记 | Skill 种子(skill_markdown) | **训练的种子**、创作 |
| **训练** | **★待建** | 种子 + 该博主笔记(train/val/test) | **best_skill.md + 提升报告** | 创作、对比卡 |
| 我的账号 | 我的账号 ✅ | 我的主页 | 我方内容缓存 | 诊断 |
| 诊断 | 对标诊断/诊断我的 ✅ | 我方内容 + 标杆 | 差距报告 + 动作清单 | **创作的"补短板"输入**、业务级指标 |
| 创作 | 对标博主创作 ✅ / 自由创作(待建) | best_skill + 差距 + 选题 | 草稿(正文/脚本/配图) | 合规、发布 |
| 合规/对标 | 创作内 critic/benchmark/compliance ✅ | 草稿 | 合规结论 + 对标差距 | 发布把关 |
| 发布 | 发布草稿 ✅ | 草稿 | 已发布内容 | **互动回流(①)** |

> 现状：除「★训练」「自由创作」「①互动回流」「③差距收敛曲线」外，其余链路已上线。本设计就是把这几块补上、并把它们接进闭环。

## 3. Skill 训练在闭环里的精确位置
- **上游**：消费「蒸馏」产出的 Skill 种子 + 「采集」的该博主笔记（切 train/val/test）。
- **训练信号（双奖励源）**：
  - **离线·笔记保真度**（PoC 阶段唯一信号）：用博主真实笔记当 gold，在 test 上量"像不像这个博主"（StyleDist/归属/判别/评委，全自动零人工，见训练文档第 4 节）。
  - **在线·真实互动**（闭环成熟后接入，回路①）：客户用这份 Skill 生成、并真实发布的内容的点赞/收藏/评论，作为**额外奖励**持续微调 Skill。离线保证"像"，在线保证"有效"。
- **下游**：best_skill 进入「创作」；训练的"前/后提升卡"进入产品，成为 skill 级可量化数据。
- **不改变形态**：训练前后都是**一个** best_skill.md，沿用现有 skill 存储 + 创作消费路径，不破坏现链路。

## 4. 创作阶段如何"串"两个输入
创作不是只吃 Skill，而是 **best_skill（怎么写得像标杆）+ 诊断差距（我缺什么）+ 选题**：
- Skill 决定"风格/结构/语言 DNA"。
- 诊断差距决定"这条要补我的哪个短板"（如"我的开篇钩子弱"→ 本条强制强钩子）。
- 这一步把"标杆侧"和"自己侧"两条线真正合流——否则就是"只会模仿、不知道补自己"。
- 现状：创作已吃 Skill；**「把诊断差距作为创作约束」是要补的接线**（让对标诊断的产出结构化地喂进创作 agent）。

## 5. 三条反馈回路（"自我改进"的来源）
1. **互动回流（内容级·快）**：发布内容的真实互动 → 既是在线奖励微调 Skill，也回灌诊断（"我哪类选题真的涨粉"）。需要：发布后定期回采我方已发内容的互动数（复用采集能力）。
2. **标杆演化（标杆级·中）**：增量采集标杆新笔记 → 笔记池更新 → 触发"重蒸/重训"建议（标杆变了，Skill 该更新）。需要：检测笔记池变化量 → 提示或自动排期重训。
3. **差距收敛（业务级·慢）**：周期性重跑「对标诊断」→ 画"我 vs 标杆差距随时间收敛"的曲线。**这是给客户看的终极证据**，也反过来验证 Skill 训练到底有没有用。需要：诊断结果留痕 + 趋势视图。

## 6. 可量化指标：分两层（都要有）
- **Skill 级（训练直接产出）**：test 集上 种子 vs best 的保真度提升（如 62→78，+16）、可辨别度下降（81%→58%）。证明"训练这一步有效"。
- **业务级（闭环产出，客户真正在意）**：
  - 差距收敛：对标诊断综合分随周期上升 / 短板项减少。
  - 互动趋势：我方发布内容的赞藏评均值/中位数走势。
  - 产能/成本：单条创作的自检修订次数下降（Skill 变强 → 推理更省）、单条成本下降。
- **两层要打通**：对客户讲故事是"训练让 Skill +16 分 → 创作更像标杆 → 30 天差距收敛 X% / 互动 +Y%"。

## 7. 现状盘点（已建 vs 待建）
| 能力 | 状态 |
|---|---|
| 采集(增量/URL/ASR/去重)、博主资产、蒸馏、诊断、创作、合规、发布草稿、任务队列、成本记账、统一进度面板 | ✅ 已上线 |
| **Skill 训练（离线·笔记保真度）** | ★ 待建（本轮重点，先 PoC） |
| 诊断差距 → 结构化喂进创作 | ⬜ 待接线 |
| 互动回流（回采我方已发内容互动 + 在线奖励） | ⬜ 待建（回路①） |
| 标杆演化触发重训 | ⬜ 待建（回路②，可半自动：提示） |
| 差距收敛曲线（诊断留痕 + 趋势视图） | ⬜ 待建（回路③，业务级证据） |
| 自由创作（不依赖对标） | ⬜ 占位 |

## 8. 落地顺序（把 Skill 训练放进整体 roadmap）
按"先验证杠杆、再接闭环、最后做业务级证据"排：
1. **M1 · Skill 训练 PoC**（离线脚本，零人工）：产出 test 集 skill 级提升数。**Gate**：提升真实且度量可信 → 继续。（细节见训练文档 Phase 0）
2. **M2 · 训练产品化**：SkillTrainingRun 模型 + RQ 训练任务 + 进度/成本 + 创作可选 best_skill + 前后对比卡。
3. **M3 · 差距→创作接线**：把对标诊断的短板结构化注入创作约束（标杆侧×自己侧合流）。
4. **M4 · 差距收敛曲线**（回路③）：诊断留痕 + 趋势视图 → 业务级可量化证据。
5. **M5 · 互动回流**（回路①）：回采我方已发内容互动 → 在线奖励持续精炼 Skill。
6. **M6 · 标杆演化重训**（回路②）：笔记池变化触发重训提示。

> M1–M2 让"训练这一步"成立且可量化；M3–M6 才把它真正接进业务闭环、产出客户在意的"涨粉证据"。**先 M1，但设计上为 M3–M6 留好接口（诊断结果结构化、已发内容可回采、Skill 可多版本）。**

## 9. 给训练文档补的一句话定位
原 [Skill训练_SkillOpt方案设计.md](Skill训练_SkillOpt方案设计.md) 是"杠杆的内部机械"；本文是"杠杆装在哪台机器上、撬动哪个业务指标"。两者配合：训练负责让 Skill 更像标杆（离线保真度），闭环负责让"更像标杆"变成"客户账号更涨"（在线互动 + 差距收敛）。
