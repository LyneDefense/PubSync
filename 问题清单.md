## #9 · 💬 待讨论 · 2026-06-19 02:48
数据源选型:除 TikHub 外的小红书/抖音数据获取方案。
- **TikHub 不开源**,是商业付费 API(14+ 平台 700+ 接口),不能自部署;作者 Evil0ctal 另维护开源的 `Douyin_TikTok_Download_API`(抖音/TikTok/快手/B站,可 Docker 自部署),但基本不覆盖小红书。
- 商业替代:EnsembleData、Apify、Bright Data/Zyte。
- 开源自部署:MediaCrawler(小红书/抖音/快手/B站/微博等,Playwright+登录cookie,有 Pro 版)、Spider_XHS、ReaJason/xhs。
- 取舍:自部署省 API 费,但要长期养登录态/IP 代理池/风控对抗/解析维护(小红书风控最硬)。倾向路径:**抖音可考虑切自部署,小红书继续走商业 API**。待评估抖音调用占比与可省成本后再定。
